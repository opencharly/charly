## 2026-07-02 — Complete the OpenCharly rename cutover: delete residual `overthink` backward-compat

The genesis rename (`v2026.183.1359`) left a small set of `overthink`-era
backward-compatibility hooks in the runtime instead of a hard cutover. Per the
hard-cutover policy (backward compatibility lives ONLY in `charly migrate`), they
are now deleted.

### Deleted (runtime backward-compat)

- `kit.StripLegacyOverthinkBlocks` (`charly/plugin/kit/profile.go`) — the shell-rc
  writer helper that stripped pre-rebrand `# overthink:begin` / `# overthink:end`
  managed blocks — plus its call in `ReplaceOrAppendManagedBlock`, the package-main
  alias in `charly/kit_aliases.go`, its two call sites in `charly/shell_profile.go`
  (`replaceOrAppendManagedBlock` / `replaceOrPrependManagedBlock`), and the pointer
  NOTE in `charly/plugin/kit/migrate_helpers.go`. Fresh installs never carried an
  `# overthink:` block; the live `# opencharly:` managed-block write/replace path is
  unchanged.

### Swept (comments only)

- `charly/refs.go` — the remote-cache migration comments (`EnsureRepoDownloaded` /
  `cacheBehindHead`) no longer name the pre-rebrand `overthink.yml` cache filename;
  the generic "no `charly.yml` → migrate" behavior is unchanged.

After this change `git grep -i overthink` returns only this `CHANGELOG/` history.

## 2026-07-03 — Ledger DeployRecord: omit empty candy (bootstrap-VM venue-record egress fix)

`DeployRecord.Candy` carried `json:"candy"` (no `omitempty`, unlike its sibling
`AddCandy`), so a nil Candy slice marshalled to `"candy": null`.
`recordVenueLedger` (deploy_target_external.go) builds a guest venue DeployRecord
with NO plan-candies for a bootstrap VM — its candy is applied via pacstrap, not
a plan step — so the guest ledger write failed egress validation
(`candy: conflicting values null and [...string]` against
`#DeployRecord.candy?: [...string]`), failing `check-arch-pacstrap-vm`'s
deploy-add step.

**Fix:** add `omitempty` to `DeployRecord.Candy`, so an empty candy list is
omitted (satisfying the OPTIONAL `candy?` schema) instead of serialising to
`null`. A populated candy list is unaffected.

Proven: `check-arch-pacstrap-vm` PASS (was FAIL on the egress conflict);
`TestValidateEgressValue_DeployRecord` + the host-infra ledger tests green.

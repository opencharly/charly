## 2026-07-03 — `box list boxes`: a project-less directory is an EMPTY list, not an error

`charly box list boxes` hard-errored `no charly.yml found in <dir>` when run in a
directory with no project. The charly-mcp `box.list.boxes` MCP tool runs exactly
that, in the coder pod's `CHARLY_PROJECT_DIR` (`/workspace`) — empty before the user
scaffolds anything — so the tool always errored (the last failing check on the coder
beds after the DNS fix).

Listing is graceful-on-empty (like `ls` in an empty dir): a project with no
charly.yml has ZERO boxes, exit 0, no output — not a failure. Added the
`ErrNoCharlyYml` sentinel (wrapped by the ONE `noCharlyYmlErr` construction shared by
config.go + format_config.go, R3) and made `ListBoxesCmd` return an empty list when
`LoadConfig` reports an absent project. A real load failure (malformed charly.yml)
still propagates.

Proven: `TestListBoxes_NoProjectIsEmptyNotError` (sentinel + graceful, FAILS pre-fix);
fresh-rebuild live — `box list boxes` in an empty dir exits 0 with no output, and in
a project still lists its boxes. (The coder beds' baked charly is pinned; the
box.list.boxes check flips green once box/<distro> bumps the charly-candy pin.)

# 2026.184.1901

## 2026-07-03 — fix: check beds are concurrency-safe (per-bed config isolation + killed-probe robustness)

Running the full disposable check-bed roster at high concurrency (maxjobs 22) surfaced a
batch of spurious failures that PASS when a bed runs alone. Three real charly root causes
(NONE fixed by throttling — reducing concurrency only HID them):

1. **Per-bed deploy-config isolation (the primary bug).** Every `charly check run <bed>`
   wrote its ephemeral deploy state (`resolved_port`, quadlet entries) to the SHARED
   operator `~/.config/charly/charly.yml`. Under concurrency the read-modify-write of that
   ONE file raced — one bed's write landed in another bed's node (`check-sway-browser-vnc-pod`
   stored as `kind:group` with a `resolved_port` `#GroupInput` forbids) → the file became
   unloadable → EVERY subsequent bed's `config`/`update` cascade-failed. Fix: a
   `CHARLY_DEPLOY_CONFIG` path override (`defaultDeployConfigPath`); the bed runner points
   each bed at a PER-BED isolated config (env-scoped + cleaned), so concurrent beds never
   share — or corrupt — the file, and a disposable bed's transient state never pollutes the
   operator's persistent config.

2. **Signal-kill was masked as `exit -1`.** `runCaptureCmd` collapsed a signal-terminated
   process (`ee.ExitCode() == -1`) into `(-1, nil)` — nilling the error — so a probe SIGKILLed
   by its per-attempt deadline under load surfaced as an ordinary "probe exit -1" check
   failure with empty stderr, undiagnosable. Now surfaced as `process terminated by signal (…)`
   (this is what exposed causes 1 and 3).

3. **Killed probes are re-attempted.** A check probe KILLED before completing produced no
   result — an infra interruption, not a check result. `runWithEventually` now auto-retries a
   killed probe (bounded by `killedProbeRetries`), distinct from a probe that RAN and returned
   a failure (never retried — not retry-on-flake).

R10: the 9 previously-failing "high-concurrency artifact" beds (addcandy, charly-fedora,
jupyter, jupyter-ml, preempt-live, selkies-labwc, sway-browser-vnc, githubrunner, doctor-local)
now pass 9/9 run ALL-CONCURRENT and UNTHROTTLED (deliberately oversaturating a 16-core host);
the operator config is untouched. Unit tests: TestRunWithEventually_RetriesKilledProbe,
TestDeployConfigPath_EnvOverride.

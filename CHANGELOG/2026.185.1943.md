# llama-cpp: select the newest release that actually ships a binary

candy/llama-cpp downloaded from `releases/latest`, which can point at a
just-tagged release whose CI has not yet uploaded the -bin-ubuntu-x64 asset —
yielding a build-time 404 (observed on llama.cpp b9871). The candy now walks the
releases list newest-first and takes the first release that actually ships the
ubuntu-x64 binary; the source tree uses the SAME release, so binaries and the
GGUF conversion tools always match. The two former download steps merged into
one (ONE release selection, no cross-step drift). R10: check-cachyos-jupyter-ml-pod
PASS (10/10, fresh rebuild).

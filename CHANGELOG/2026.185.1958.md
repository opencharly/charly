# container-nesting: bind the outer /proc so nested podman works with a GPU

Nested rootless podman failed with `crun: mount proc to proc: Operation not
permitted` on any image composing container-nesting AND a GPU CDI device
(nvidia.com/gpu=all): the nvidia container toolkit injects a real
/proc/driver/nvidia/params submount, and the kernel's mount_too_revealing check
then refuses the inner fresh /proc mount EVEN WITH unmask=/proc/* (unmask removes
masked paths, not submounts). The containers.conf now binds the outer /proc into
every nested container (bind-propagation=rslave), avoiding the fresh mount —
nested podman works with or without a GPU. Verified: nested run + nested build,
GPU + no-GPU. R10: check-fedora-coder-pod PASS (10/10, nested-podman-run exit=0,
fresh rebuild).

# charly remove --purge: also purge encrypted (gocryptfs) volumes

`charly remove --purge` removed named podman volumes but left the gocryptfs
cipher/plain dirs behind (an encrypted volume is a filesystem dir under the
encrypted storage path, not a podman volume). A purged disposable enc bed thus
left an orphaned, credential-less cipher dir, and the next deploy failed to mount
it ("cipher: message authentication failed"). --purge now also removes the
deploy's encrypted volume dirs, enumerated by the `charly-<storageDir>-` prefix
under the encrypted storage path (works even when the deploy config is already
gone — the orphan-after-crash case), unmounting each FUSE plain dir first. R10:
`charly remove --purge` removes the cipher dir (Removed encrypted volume enc-data
→ PURGED-OK); check-enc-pod bed PASS (10/10, enc deploy path unaffected).

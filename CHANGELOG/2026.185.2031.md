# pac localpkg install: refresh the sync DB before installing the local package

The pac localpkg install_template ran `pacman -U <pkg>` without refreshing the
sync DB, so a local package's depends= resolved against the pinned base image's
stale DB. As the base image ages, its dep versions rotate off the mirrors —
`pacman -U opencharly-git.pkg.tar.zst` then 404'd on skopeo-1.23.0-1.1 (and
podman/netavark/aardvark-dns), which every cachyos mirror had dropped. The
install_template now reseeds the keyring + `pacman -Sy` first (the pacman analog
of the deb template's `apt-get update`), kept a single &&-chained shell command
so the localpkg renderer wraps it in one RUN. R10: check-githubrunner-pod PASS
(10/10, fresh rebuild) — the skopeo 404 that reproduced on every mirror is gone.

# notebook-osm: fix the shortbread tilemaker conversion (bbox + skip-integrity)

The osm-monaco-viz notebook's shortbread DAG (tilemaker → monaco-shortbread.pmtiles)
failed while every other tile pipeline succeeded, so the pmtiles was never
generated and the versa tile/pmtiles checks failed ("No such file"). Two tilemaker
requirements were unmet: (1) the shortbread config.json has shapefile-backed
layers (ocean/land polygons), and tilemaker refuses to read shapefiles without an
explicit --bbox; (2) the downloaded monaco extract is CLIPPED, so boundary ways
reference nodes outside the cut and tilemaker aborts with
`std::out_of_range: Could not find node with id N` without --skip-integrity. Added
both flags (MONACO_BBOX covering the extract). Also corrected the wait-for-parquet
comment, which mis-attributed the shapefile error to a mid-replace race. R10:
check-versa-pod PASS (10/10, fresh rebuild) — the shortbread pmtiles is generated
and the tile checks pass.

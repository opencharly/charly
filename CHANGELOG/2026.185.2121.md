# M1: every compiled-in kit check verb is now dual-placement (cmd/serve shims)

The 13 kit check-verb candies (addr, command, dns, examplerunverb, file, interface,
kernel-param, mount, package, process, service, unix-group, user) already exported
the importable `NewCheckVerb()` + `SchemaFS`/`SchemaDir`/`InputDefs` and compiled INTO
charly via compiled_plugins, but they shipped no `cmd/serve` entrypoint — so they could
run ONLY compiled-in, never out-of-process. Added the thin `cmd/serve/main.go` shim
(the plugin-port pattern: `sdk.ServeCheckVerb(pkg.NewCheckVerb(), ver, pkg.SchemaFS,
pkg.SchemaDir, pkg.InputDefs)`) to each, completing M1: every compiled-in kit verb is
now dual-placement (compiled-in vs out-of-process is a per-build choice, never an
authoring one). `go mod tidy` (GOWORK=off) synced each candy's go.sum to sdk's full
transitive dep graph (zero version drift vs sdk/go.mod). The compiled-in path is
byte-identical — pluginsgen reproducible, `task build:charly` unchanged. R10: all 13
serve binaries build standalone (GOWORK=off); check-pod PASS (10/10, fresh rebuild) —
the compiled-in kit verbs (port/file/command/process/service) still pass.

# Plugin-shape unification: all 34 package-main plugins → importable Shape A

Every plugin candy is now the ONE unified dual-placement shape: an importable root
package exporting `NewProvider()`/`NewMeta()` (+ `CliMain()` for command plugins),
with a thin `cmd/serve/main.go` shim serving it out-of-process. The 34 plugins that
still hand-rolled `package main` at their root — 10 verb (adb, appium, cdp, dbus,
kube, mcp, record, spice, vnc, wl), 9 command (candy, clean, doctor, feature,
secrets, settings, tmux, udev, vm), 4 builder, 3 deploy, 8 example — were converted:
`main.go` → `plugin.go` (package renamed, `func main()` deleted, constructors
exported), a `cmd/serve` shim added. `package main` at a plugin root is now abolished;
placement (compiled-in vs out-of-process) is purely a per-build choice, never an
authoring one. The compiled-in path is byte-identical (none of the 34 are in
compiled_plugins; pluginsgen reproducible; `task build:charly` unchanged); the host
already auto-detects `./cmd/serve` as the out-of-process build target
(plugin_loader.go), and the arch PKGBUILD + the deb/rpm localpkg build_templates were
taught the same auto-detect (a Shape A root builds to a non-exec .a archive).

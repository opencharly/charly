# Canonical-go.mod gate + `task plugins:tidy` — zero-drift for the 74-module surface

Every plugin candy is its own Go module (the deliberate dependency-shed design). Two
mechanisms now keep that surface drift-free. (1) The canonical-go.mod GATE — a Go test
(charly/plugin_gomod_canonical_test.go) asserting every candy/plugin-*/go.mod matches
the Shape-A template: `module .../candy/<dir>`, `require github.com/opencharly/sdk
v0.0.0`, the single `replace => ../../sdk`, and NO dependency on the charly core module
(the Cutover-1 invariant — a plugin imports ONLY the SDK). The plugin's own deps are
free to vary; plugin-spice's extra third_party replace is the sanctioned outlier.
(2) `task plugins:tidy` (taskfiles/Plugins.yml) — sweeps `GOWORK=off go mod tidy` across
all plugin modules to keep each go.sum in step with its imports. Verified: the gate
passes across all candy modules; `task plugins:tidy` tidies 74 modules with zero drift.

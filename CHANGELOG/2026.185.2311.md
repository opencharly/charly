# NewMeta dedup: 58 plugins collapse their hand-rolled meta+Describe into sdk.NewMeta

Every plugin candy carried a byte-identical `type meta struct{ pb.UnimplementedPluginMetaServer }`
plus a `Describe` body forwarding to `sdk.BuildCapabilities(calver, caps, schemaFS, "schema")`. The new
`sdk.NewMeta(calver, caps, schemaFS)` helper (sdk v0.2026185.2311) collapses all 58 into ONE shared
implementation (R3): each plugin's `NewMeta()` is now a single `return sdk.NewMeta(calver, caps,
schemaFS)` carrying only its genuinely per-plugin data. Behavior is identical (NewMeta's Describe calls
the same BuildCapabilities); the NewMeta SIGNATURE is unchanged, so the compiled-in plugins_generated.go
path is unaffected (pluginsgen reproducible). Grep self-test: zero `type meta struct` /
`func (meta) Describe` / `sdk.BuildCapabilities(` calls remain in any plugin root.

# gofmt: normalize import ordering in 6 plugin roots

Behavior-neutral formatting fix. candy/plugin-preempt/arbiter.go's import order was left
un-normalized after the NewMeta migration's gofmt pass wasn't staged; candy/plugin-{command,
file,http,kernel-param,mount}/plugin.go carried a pre-existing gofmt divergence (import
grouping) that predated this session on origin/main. gofmt import reordering produces
identical compiled output, so the affected plugins (kit verbs, proven by check-pod; the
preempt arbiter, proven by check-preempt-local's resolve+invoke) are unaffected. The entire
candy/plugin-* tree is now gofmt-clean.

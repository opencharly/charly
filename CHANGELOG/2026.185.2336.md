# Modifier dedup: 10 live-verb plugins drop modifierZero for sdk.RequireModifiers

The 10 live-verb plugins (cdp/vnc/wl/dbus/mcp/record/spice/adb/appium/kube) each hand-rolled a
`modifierZero(op, name)` func (a modifier-name → spec.Op field zero-check) plus a per-plugin
required-modifier loop or wrapper. All are deleted in favour of sdk.RequireModifiers (sdk
v0.2026185.2336), which uses the generic reflection-based sdk.OpModifierZero — one implementation
serving every verb (R3). Each plugin keeps ONLY its requiredModifiers map (genuinely per-verb data).
The replacement is authorized by the SDK's TestOpModifierZeroMatchesFields differential-equivalence
gate. Stale test references to the deleted funcs were repointed at sdk.RequireModifiers (coverage
preserved). Grep self-test: zero `func modifierZero` / `func checkRequiredModifiers` remain.

out-of-process), check-k3s-vm PASS (kube), both fresh rebuild. spice/adb/appium proven-by-form.

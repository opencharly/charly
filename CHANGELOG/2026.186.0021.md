# ServeCheckVerb collapse: 15 kit candies get NewMeta — one authoring shape for every plugin

The kit check-verb candies were the last plugins with a non-unified meta: they exported a
SchemaFS/SchemaDir/InputDefs trio consumed by the old 5-arg sdk.ServeCheckVerb. That trio is gone.
sdk.ServeCheckVerb collapses to (kv, meta) (sdk v0.2026186.21); each of the 15 kit candies (port,
addr, command, dns, examplerunverb, file, http, interface, kernel-param, mount, package, process,
service, unix-group, user) now exports its OWN NewMeta() (sdk.NewMeta with explicit caps + its
embedded schema) — the SAME NewCheckVerb()/NewProvider() + NewMeta() shape as every pb-provider
plugin (R3). charly's registerCompiledCheckVerb(kv, meta) reads the schema + input-defs from the
candy's NewMeta via Describe (mirroring registerCompiledPlugin's meta-driven in-proc lift); pluginsgen
emits registerCompiledCheckVerb(NewCheckVerb(), NewMeta()). The exported SchemaFS/SchemaDir/InputDefs
trio is deleted (SchemaFS unexported; SchemaDir/InputDefs's data lives in NewMeta). Grep self-test:
zero `var SchemaFS` / `const SchemaDir` / `var InputDefs` remain in any kit candy.

With this, EVERY plugin candy — pb-provider and kit — has the one unified authoring shape. R10:
check-pod PASS (10/10, fresh rebuild) — the compiled-in kit verbs (port/file/command/process/service)
register + run via NewMeta; all 15 kit serve binaries build standalone; pluginsgen reproducible.

# M15: migration engine externalized to candy/plugin-migrate (compiled-in command:migrate)

The config-schema MIGRATION ENGINE moved OUT of charly core into a new compiled-in command plugin
`candy/plugin-migrate` (command:migrate — the plugin-preempt shape: NewProvider/NewMeta/CliMain +
cmd/serve). The CUE-anchored declarative table + generic op-walker + file-walk drivers (formerly
charly/migrate_engine.go + migrate.go + migrate_cmd.go, ~600 LOC) live in the plugin now; the
migration DATA (migrations.cue) is embedded there; #Migration is read from the SDK schema
(sdk/schema — migration.cue + version.cue, since #Migration.version pins to the SDK's single-source
#CanonCalVer, which never leaves sdk/schema).

Core keeps ONLY the kernel seams:
- `charly migrate` dispatches to the compiled-in command:migrate (main.go's Migrate MigrateCmd field
  deleted; the word arrives dynamically like preempt/vm).
- refs.go remote-cache auto-migration resolves command:migrate + Invoke(OpRun {--project-only
  --quiet --dir <cache>}) in-proc (autoMigrateCacheProjectOnly) — replacing the direct
  RunProjectMigrations call. command:migrate is compiled-in, so it resolves at init(), available
  deep in config loading before LoadUnified completes.
- The below-floor/behind-head load-gate HINTS (the "Run: charly migrate" messages) stay core.

Shared helpers moved to kit (R3): DeployConfigPath → kit.DefaultDeployConfigPath (core var alias +
plugin caller); mappingRoot → kit.MappingRoot (core alias in kit_aliases.go + plugin engine). The
dead opUnifyCandidateFiles core alias deleted (migrate was its sole consumer). Only
migrate_engine_test.go moved to the plugin; the six migrate_*_test.go core loader-gate tests stay
(they test LoadUnified/VmDeployState/etc., not the engine).

R10: the new disposable check-migrate-local bed (a kind:local deploy composing plugin-migrate) proves
the compiled-in command:migrate resolves + Invoke(OpRun) runs the engine end-to-end on a fresh
rebuild — at-HEAD no-op (exit 0), --dry-run plumbing (exit 0), below-floor refusal (exit 1) — PASS
(steps=4). The declarative transform ops are covered by the plugin's engine_test.go units. bumps:
sdk → v0.2026186.124 (kit.DefaultDeployConfigPath + kit.MappingRoot); plugins → M15 skill sweep.

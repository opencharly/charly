# M4a: pod substrate deploy lifecycle externalized out of core (candy/plugin-deploy-pod)

The POD deploy substrate's host-side venue LIFECYCLE moved OUT of charly core into the external
candy/plugin-deploy-pod (M4a — the pod half of M4; vm remains compiled-in for M4b). The compiled-in
podSubstrateLifecycle (charly/pod_deploy_lifecycle.go, deleted) is replaced by the plugin serving the
substrate-lifecycle Ops over the reverse channel, so pod is now externalized like local/vm/k8s/
android — its lifecycle no longer branches in core.

The generic seams M4 adds (all F11-clean — generic host-builders/wire, never provider words):
- grpcSubstrateLifecycle proxy Invokes every lifecycle Op WITH the host executor
  (InvokeWithExecutor), ships HostEnv{CharlyBin,Home} on op.Env + spec.LifecycleOpts in params,
  decodes spec.PrepareVenueReply (venue + opaque State patch the host persists via saveDeployState +
  Notes it prints) / spec.PostTeardownReply.
- A new generic "cli" F10 host-builder (charly/host_build_cli.go): the plugin asks the host to run a
  `charly <argv>` subcommand (inherit stdio for interactive legs / capture for a status parse) —
  replacing the in-core runCharlySubcommand/captureCharlyStdout the compiled-in pod used.
- The live overlay-build inputs (compiled plans + nested parent venue) thread ctx → reverse-server
  (executorReverseServer.live) → the "overlay" builder, so HostBuild("overlay") re-attaches them
  host-side (they cannot cross the []byte wire).
- The overlay BUILD engine STAYS core in charly/build_overlay.go (runOverlayBuild + PodDeployTarget/
  OCITarget/Generator — never cross the module boundary); the plugin reaches it via HostBuild.
- A hidden `charly __drop-overlay <name>` command the plugin's PostTeardown forwards to for the
  host-side overlay-image drop.

candy/plugin-deploy-pod now declares Lifecycle:true (so registerPluginSubstrateLifecycle wires pod's
lifecycle at plugin-load) and serves the full lifecycle (PrepareVenue via HostBuild("overlay"),
Start/Stop/Status/Logs/Shell/Rebuild/PostTeardown via HostBuild("cli"); OpExecute still walks
nothing). plugin-example-lifecycle updated to the PrepareVenueReply reply.

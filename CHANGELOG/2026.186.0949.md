# refactor(settings): externalize the charly settings command into candy/plugin-settings (compiled-in)

The `charly settings` command LOGIC leaves core. candy/plugin-settings is now a COMPILED-IN command
plugin (command:settings) that OWNS the command — the get/set/list/reset/path subcommand grammar + the
output formatting. Its Invoke(OpRun) reaches the SHARED config subsystem over the generic "settings"
HostBuild seam (host_build_settings.go, sdk v0.2026186.948's SettingsRequest/Reply).

The config subsystem (GetConfigValue / SetConfigValue / ListConfigValues / ResetConfigValue /
RuntimeConfigPath + the credential store + engine/runtime resolution) STAYS core (runtime_config.go).
The get-resolution special-cases (engine.* via ResolveRuntime, secret_backend via the credential store,
vnc.password.*/hosts.* via GetConfigValue) moved to resolveSettingsGet (host_build_settings.go).

+ the __settings main.go field. The out-of-process cliMain path errors (no reverse channel) — settings
is compiled-in (reuses Seam A landed with command:clean).

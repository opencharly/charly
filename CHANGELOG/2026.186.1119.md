# refactor(doctor): externalize the charly doctor command into candy/plugin-doctor (compiled-in)

The `charly doctor` command LOGIC — the WHOLE 910-line host-dependency report — leaves core.
candy/plugin-doctor is now a COMPILED-IN command plugin (command:doctor) that OWNS the entire report:
the check list, group orchestration, pass/warn/fail verdicts, human + JSON formatting, exit code, AND
the pure host ops it runs itself (binary probes via exec.LookPath, /etc/fuse.conf + config-permissions
reads). vmChecks uses sdk/vmshared directly.

The plugin reaches ONLY the genuine host-hardware subsystem it can't itself hold — over ONE generic
"hostprobe" HostBuild seam (host_build_hostprobe.go, sdk v0.2026186.1117's HostProbeRequest/Reply): the
GPU/VFIO/device detection primitives (the existing C11 shims, multi-caller with deploy/vm), credentialHealth
(verb:credential, which lazy-connects host-side), and the core install-hint/device-description tables. The
seam returns RAW FACTS ONLY — zero report/verdict logic crosses into core.

spec.CredentialHealth moved to sdk (core aliases it, R3). DELETED: charly/doctor.go (+ doctor_test.go)
and the __doctor main.go field. deviceDescriptions relocated into the seam file (still core data).

R10: check-doctor-local PASS (steps=4) exercising `charly doctor` (+ --json verified) through the new
compiled-in→HostBuild("hostprobe") path — GPU/VFIO/secret/device sections all render from the seam.
bumps sdk v0.2026186.1117 + plugins.

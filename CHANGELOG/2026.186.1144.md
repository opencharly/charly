# refactor(feature): externalize the charly feature command into candy/plugin-feature (compiled-in)

The `charly feature` command (list / pending / validate) LOGIC leaves core. candy/plugin-feature is now
a COMPILED-IN command plugin (command:feature) that OWNS the subcommand grammar + the output formatting.
Its Invoke(OpRun) reaches the genuine core subsystem it can't hold — the unified LOADER (LoadConfig /
ScanCandy — the kernel), the Step plan model, and validatePlanSteps (shared with `charly box validate`,
R3) — over the generic "feature" HostBuild seam (host_build_feature.go, sdk v0.2026186.1143's
FeatureRequest/Reply), which enumerates every entity's plan into plain DATA.

DELETED: charly/feature_internal_cmd.go (FeatureList/Pending/ValidateInternalCmd + the render* funcs) +
the 3 __feature-* main.go fields. validatePlanSteps moved to charly/plan_validate.go (still core, shared
with box validate). The now-false TestExternalCommandExecPlan_Feature (feature is compiled-in, not
syscall.Exec'd) removed — the exec-plan mechanism is still covered by TestExternalCommandExecPlan_Vm.

R10: check-feature-local PASS (steps=4) exercising `charly feature list candy` (+ list/pending/validate
verified live) through the new compiled-in→HostBuild("feature") path. bumps sdk v0.2026186.1143.

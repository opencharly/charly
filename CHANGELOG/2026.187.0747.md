# Schema compaction — the compact node grammar, plugin-owned verb schemas

One hard cutover; schema `2026.186.2323` (floor stays `2026.174.1100`).

## The compact authored grammar

- An entity is `<name>: {<kind>: <full body>}` — collections (`env:`, `volume:`,
  `port:`, `service:`, `secret:`, deploy data like `add_candy:`/`install_opts:`/
  `vm_state:`) live INLINE in the kind value. The former named data-child nodes
  (`<entity>-<key>: {<key>: …}`) and named/auto-keyed step-child nodes
  (`<entity>-step-N:`, `my-step:`) are load errors pointing at `charly migrate`.
- Steps are an ordered, unnamed `plan:` list inside the kind value. A former
  meaningful step-child key became that step's `id:`.
- A step carries ONE intent keyword (`run:`/`check:`/`agent-run:`/`agent-check:`/
  `include:`) plus at most one verb key: a builtin install verb (`mkdir`/`copy`/
  `write`/`link`/`download`/`setcap`/`build`) or the generic plugin sugar
  `<word>: <input>` — map value = the plugin input verbatim; scalar value =
  `{<primary>: value}` via the verb's declared primary. Authoring `plugin:` +
  `plugin_input:` in a step is a hard load error (internal-only envelope).
- Member children (sub-entities under a deploy) stay named — member names are
  load-bearing and must not collide with kind words.
- `env:` is a map everywhere; the box/deploy `KEY=VALUE` list form is gone.
- Corpus effect: ~14% smaller authored YAML at full functionality.

## Plugin-owned verb schemas

- The 11 live verbs (cdp/wl/dbus/vnc/mcp/record/spice/libvirt/kube/adb/appium)
  moved their method enums + ~70 exclusive `#Op` fields into their plugins'
  input schemas; each declares `primary: method`. Renames: dbus `method`→`member`
  (the D-Bus member), cdp/appium/libvirt HTTP-method→`http_method`; http absorbed
  `method`/`request_body`; appium absorbed `caps`; wl absorbed `target`+`command`.
- Verb primaries: `ProvidedCapability.Primary` (proto field 10) + the candy
  manifest `plugin.primary` map (prescanned pre-parse for external plugins);
  `registerPluginUnitSchema` cross-checks the primary against the served input
  def; provider registration rejects a verb word colliding with an authored
  `#Op` field.
- The stub-schema requirement is gone: an input-less plugin ships no schema.
  20 gate-only stub schema dirs deleted.

## Deletions (user-approved)

- The Calamares trio: `target:`/`module:`/`package-group:` kinds,
  candy/plugin-target, plugin-module, plugin-package-group, check-target-layer,
  and the box/fedora check-target bed. Returns as its own cutover when the
  installer feature is real. (`group:` — the targetless deploy group — remains.)
- The never-authored `#Op` harness surface: `summarize`/`kill` verbs, `signal`,
  `over_id`/`metric`/`emit_id`/`p50`/`p95`/`p99`/`max`/`mean`, `capture`/
  `capture_extract`, `parallel`/`count`/`index_var`, plus their runner code.
- Audit verdicts: `#Candy.engine` RETAINED (live per-image engine resolution +
  conflict validation); `#Candy.info` never existed in the schema (stale docs
  swept).

## Migration

- The first real `goHooks` migration: `compact-node-form`
  (`candy/plugin-migrate/reshape_compact.go`, `touches_host: true`) — frozen
  vocabulary, duplicate-data-key and unclassifiable-child fail loudly, non-manifest
  YAML gated per document. Proven on the full 312-file corpus (semantic
  losslessness + byte idempotency) before shipping.
- Remote caches auto-migrate on fetch (`EnsureRepoDownloaded`); the floor stays
  `2026.174.1100` so pinned `@github` refs keep resolving.

## Fixes surfaced by the cutover

- `#PluginCapability` gained the `command` class + underscore words (schema
  catching up with pre-existing providers).
- The `#CandyValue` concrete validate gate now runs on the parsed+desugared
  tree, scoped to layer manifests.
- The stale-binary freshness guard no longer counts `_test.go` files.
- check-group-layer restored (deploy-group bed member, wrongly swept with the
  Calamares deletion).
- The `default` bed's dual-verb vm-guest steps (a pre-existing silent defect)
  are structurally impossible under the one-verb-key grammar.
- **Duplicate top-level entity names are now a hard load error** — `parseNodeTree`
  gained a name-collision gate (yaml.Node keeps duplicate keys as separate pairs,
  so a same-document duplicate of a different kind previously loaded BOTH
  silently, violating the global-uniqueness Key Rule).
- **Calamares deletion completed in box/fedora + plugin-vm** — the trio removal
  had left `check-target-layer` referenced by `box/fedora/box/check-target` and
  the three Calamares plugins listed in `candy/plugin-vm/build_defaults.yml`;
  both removed (the R5 grep swept Go identifiers but not cross-repo YAML
  candy-names — the bed gate caught it).
- **check-migrate-local at-HEAD fixture bumped** to `2026.186.2323` — the bed's
  "no-op at head" check used the pre-cutover version where `#SchemaFloor ==
  #SchemaVersion`; the schema bump correctly made that config migrate.

## Concurrency fixes (surfaced by a max-parallel bed roster, all RCA'd + fixed)

Running the disposable roster at maximum parallelism surfaced defects invisible
to every serial run; each was RCA'd from the actual error and root-cause fixed:

- **The `discover` walk no longer aborts the whole load on a per-entry
  `EACCES`.** `findEntityDirs` (`charly/unified.go`) now skips an unreadable /
  build-artifact subtree (`filepath.SkipDir`) and skips `.git`/`.build`/`output`/
  `node_modules` by name — a manifest can never live in an unreadable dir. Under
  concurrency a sibling bed's makepkg artifact (`pkgbuild/pkg`, fakeroot-owned)
  made every concurrent `LoadUnified` fail; `findLocalSpec` now surfaces the real
  load error instead of a misleading "unknown template".
- **The `libvirt` verb's input def is `#LibvirtVerbInput`** (was `#LibvirtInput`)
  — it collided with the base VM `<input>`-device def `#LibvirtInput`, so the
  concat leaked the base's required `type` field and broke every libvirt verb
  dispatch.
- **Every eval VM bed auto-allocates its SSH host port** (`ssh.port_auto`) — the
  fix for a real host-port collision (two beds pinned SSH `12227`); the mandate
  now lives in CLAUDE.md + `/charly-internals:agents`.
- The `.claude/workflows/verify-beds.js` discovery prompt updated for the compact
  grammar (a disposable substrate node, not the retired `kind: check`).

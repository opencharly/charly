# Plugin-first as a standing order ‚Äî the kernel/plugin boundary law

Documentation-class cutover. Codifies the kernel/plugin architecture as a binding
standing order and sweeps the stale comments that contradicted it. No behaviour
change; comment-only `.go` edits + CLAUDE.md + skills.

## The boundary law (new CLAUDE.md pillar)

- New pillar **"The kernel/plugin boundary law"**: the kernel is a kind-blind
  execution substrate holding only kind-AGNOSTIC things ‚Äî **(E)** generic
  Envelopes (`#Node`/`#Step`/`#Op`/the `#Deploy` tree/the `InstallPlan` IR + the
  wire replies), **(M)** generic Mechanisms (dispatched by word, never branching
  on a kind, irreducibly on the spine), **(B)** the single Bootstrap `candy`‚äª`box`
  root, and **(D)** kind-recognition Data. Everything concrete-kind ‚Äî schema,
  typed shape, validation, behaviour ‚Äî is a **(R)** plugin that RESOLVES into a
  generic envelope (`builder`'s `OpResolve`‚Üí`BuilderResolveReply` is the
  reference). A `spec.<Kind>` field-read, a kind-word `switch`, or a per-kind Go
  map is an **incomplete seam** ‚Äî a bug fixed immediately (one cutover each),
  never a kept exception. **Difficulty never decides placement.**
- The "Core is the kernel" Key Rule now points at the law and the incomplete-seam
  mandate instead of the soft "classified candidates, never an indefinite TODO"
  framing. Full detail lives in `/charly-internals:plugin` "The kernel/plugin
  boundary law" (placement table, decision procedure, resolve-to-envelope
  worked example, incomplete-seam catalog).

## Doctrine sharpenings (CLAUDE.md)

- **RDD / The spike:** the spike's single purpose is to discover **HOW**; its
  deliverable is the discovered mechanism, never a proven/refuted verdict. A spike
  is a support tool, NEVER an excuse ‚Äî it decides neither *whether the work is
  done* (R10) nor *whether it happens* (the plan, an immutable CONTRACT); a
  "refute" means find a different HOW, and a genuine scope-change is STOP-and-ask.
- **Hard Cutover by Default:** a **forbidden-excuse catalog** ‚Äî "it's
  complicated / too difficult / too hard / too risky / touches too much / would
  recreate X / the spike showed it's hard" ‚Äî the third named forbidden-framing
  set alongside R1's flake/transient ban and R2's out-of-scope/follow-up ban.

# Cutover D — de-type the `sidecar` kind (resolve-to-envelope)

The kernel/plugin boundary law: a concrete kind's business logic lives in its
plugin; the kernel consumes a resolved envelope. Before this, `charly config`
did the sidecar merge + CLI env-flag routing + volume/secret-name + env_from
rendering INLINE over the typed `spec.Sidecar` — an incomplete seam. Now
candy/plugin-sidecar owns all of it behind an `OpResolve` leg.

## What moved

- **candy/plugin-sidecar** gains `resolve.go` + an `OpResolve` case: it routes CLI
  `-e` flags to sidecars (app-only env returned), merges embedded < project <
  deploy templates, and resolves volume/secret names + renders each secret's
  `env_from` — all the logic deleted from `charly/sidecar.go`.
- **Kernel stores sidecar defs OPAQUELY.** `Config.Sidecar` / `BundleConfig.Sidecar`
  / `DeployStateInput.Sidecar` / `spec.Deploy.Sidecar` are now
  `map[string]json.RawMessage`. `config_image.go`'s `resolveSidecars` calls the
  plugin via `providerRegistry.ResolveKind("sidecar") → Invoke(OpResolve)` and
  adapts the reply into the host's `ResolvedSidecar` (unchanged quadlet-gen shape).
- **R5 sweep:** delete the in-core `Sidecars()`/`Merge`/`Resolve`/`SidecarEnvKey`/
  `renderSidecarEnvFrom`/`EmbeddedSidecarTemplates` + the `SidecarDef`/`SidecarSecret`/
  `SidecarVolume` aliases; move their tests to `candy/plugin-sidecar/resolve_test.go`;
  the kept host tests read opaque bodies.
- **New bed `check-sidecar-pod`** (root charly.yml): a disposable pod deploy of the
  minimal fedora `check-k8s-deploy-app` with a `mypause` sidecar attached — proves
  the resolution end-to-end (a broken de-type fails `charly config` at deploy).

# Cutover E — de-type the `agent` kind (resolve-to-envelope)

The AI-CLI grader catalog (kind:agent) — how the check/iterate harness launches
claude/codex/… — no longer lives in the kernel as a typed `spec.Agent` map. The
name-selection + default application (the former `ResolveAgent`) moves into
candy/plugin-agent's `OpResolve` leg, and the kernel stores agent bodies OPAQUELY
and consumes a generic `spec.AgentExecSpec`.

## What moved

- **candy/plugin-agent** gains `resolve.go` + an `OpResolve` case: name-selection
  (sole entry / `--agent`), the no-agents/not-found/multiple errors, and default
  application (Timeout, PromptVia) → `AgentExecSpec`.
- **Kernel opaque storage + resolved-envelope consumption**: `uf.Agents()` deleted;
  `agent_config.go`'s `resolveAgentViaPlugin` dispatches via
  `ResolveKind("agent") → Invoke(OpResolve)`. The harness (`check_loop.Agent`,
  `RunAgentOnce`, `CaptureVersion`, `resolveGraderAgent`, `check_synccreds`,
  `check_runlocal`) consumes `*spec.AgentExecSpec`; `PrintAgents` +
  `validateIterateBed` peek the opaque bodies. The `AgentConfig`/`ResolveAgent`/
  `ErrNoAgents`/`SortedAgentNames` core surface + the `AgentConfig` alias are deleted.
- **R5**: the `ResolveAgent` tests move to `candy/plugin-agent/resolve_test.go`; the
  kernel tests read the resolved envelope / opaque bodies.
- **New bed `check-agent-local`** (root charly.yml): a disposable kind:local deploy
  whose plan pairs a deterministic `check:` with an `agent-check:` step.

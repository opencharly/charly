# Cutover G — de-type the `resource` kind (resolve-to-envelope)

The `resource` kind (an exclusive-host-resource claim — a GPU vendor selector) is no
longer consumed by the kernel's GPU arbiter as the typed `spec.Resource`.
candy/plugin-resource projects it into a `ResolvedResource` via `OpResolve`.

## What moved

- candy/plugin-resource gains an `OpResolve` config leg: `spec.Resource` →
  `spec.ResolvedResource` (the gpu vendor selector).
- The kernel's arbiter/GPU consumers (`requiredGPUResource`, `gpu_imply`,
  `validateResourceDefs`, vm auto-allocate, `gatherResources`) now read
  `*ResolvedResource`, resolved via `resolveResources`
  (`ResolveKind("resource") → Invoke(OpResolve)`); resource bodies stay opaque in
  `PluginKinds["resource"]`.
- **Finalize**: `Resources()` accessor deleted; the `ResourceDef` + `GpuSelector`
  aliases removed. No charly `*.go` imports `spec.Resource`.

## Scope note — the GPU-attach split folds into Cutover L

The plan paired G with "split GPU arbiter (attach → vm/pod plugin)". The attach leg
`vfioGpuToHostdevs` produces `LibvirtHostdev` (= `spec.Libvirt`, the vm kind's model),
so moving *only* the conversion does not remove `spec.Libvirt` from core — that
happens when the vm model moves in Cutover L. The attach split is therefore
relocated into L (not dropped): G's coherent, independently-landable deliverable is
the resource de-type (`spec.Resource` removed from core), with the arbiter ledger
staying kernel as designed.

# Cutover H (part 1) — venue-hop descent-descriptor

The deploy chain's venue-hop selection (`appendHopForFlatPath`) and the check
runner's root-executor choice no longer switch on the substrate kind word — they
read a loader-stamped, generic descent-descriptor.

## What moved

- Every deploy node carries a loader-DERIVED `Descent` (`{transport, host_rooted}`).
  `LoadUnified` stamps the whole `uf.Bundle` tree ONCE via `stampBundleDescents` →
  `kit.StampDescent` (deploy_tree.go), AFTER all structural kinds fold — so every
  substrate / group / **custom** structural kind gets a descriptor without each
  plugin remembering to stamp. The word→transport MAPPING lives in `sdk/kit`.
- `appendHopForFlatPath` switches on `node.Descent.Transport`
  (`none`/`container-exec`/`ssh`/`reject`) — the generic nesting-transport, never
  the kind word. A nil descriptor is a loud error (R4), not a silent fallback.
- `check_runner_live.go` reads `node.Descent.HostRooted` instead of comparing the
  target to `"local"`.
- Persist: `descent` is stripped from the per-host overlay body (`dropMappingKey`)
  and re-stamped on reload — a derived field is never round-tripped;
  `isAutoVmDeployEntry` ignores it.

## Scope

This lands part 1 of the plan's H row — the venue-hop descent-descriptor. The `↳`
sub-row (the check-verb endpoint seam: cdp/vnc/mcp/spice/kube preresolvers → one
generic host-endpoint reverse-leg) is a separable change with its OWN R10 beds
(vnc/spice/kube) and lands as the immediate-next cutover. The status-display
`nestedChildKind` still classifies by word (a reporting path, not the venue-hop
spine); it clears with the substrate de-types (I–K) before the Cutover N gate.

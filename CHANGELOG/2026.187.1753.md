# Cutover M — de-type the `distro` build vocabulary (resolve-to-envelope; the long pole)

The `distro:` build vocabulary is no longer consumed by the kernel's build engine as
the typed `spec.Distro`. candy/plugin-distro projects it into a `ResolvedDistro` via
`OpResolve`; the build engine reads the envelope, never `spec.Distro`.

## What moved

- candy/plugin-distro gains an `OpResolve` config leg: `spec.Distro` →
  `spec.ResolvedDistro` (field-copy + Raw).
- `uf.Distros()` now resolves via the plugin (`resolveDistros` →
  `ResolveKind("distro") → Invoke(OpResolve)`) instead of decoding `spec.Distro`
  host-side; distro bodies stay opaque in `PluginKinds["distro"]`.
- The `DistroDef` alias is now `spec.ResolvedDistro` (one flip in
  `vmshared/spec_aliases.go`), so all 97 build-engine consumers — generate.go,
  tasks.go, format_config.go, install_build.go, install_plan.go, the deploy targets —
  read the envelope unchanged; `ResolvedDistro` mirrors `spec.Distro`'s fields + the
  `PrimaryFormat`/`LocalPkgFormat` methods. No charly `*.go` imports `spec.Distro`.
  Distro is a flat plugin kind (no host pre-decode residual, unlike the substrate
  templates).

## Verification (fully tested and validated)

`go test ./...` green — `Distros()`-consuming tests drive `resolveDistros` via the REAL
plugin `OpResolve` dispatch; the new `TestResolveDistro_FieldCopyAndMethods`
(plugin-distro) proves the field-copy + the mirrored `PrimaryFormat` (secondary-format
skip); gen-repro idempotent (hand-written wire); fresh `task build:charly`;
`charly box validate` resolves the distro vocab via the plugin live. R10 (the distro
matrix, all three format families, concurrent): **fedora** (rpm), **debian** (deb),
**arch** (pac) box builds each regenerated a fresh Containerfile via `resolveDistros`
→ `ResolvedDistro.Format[fmt]` with the correct install directives
(3×`dnf` / 3×`apt` / 2×`pacman`) and built to a versioned image — byte-equivalent
rendering (stable content-version). ubuntu/cachyos share the deb/pac paths. No
`#SchemaVersion` bump.

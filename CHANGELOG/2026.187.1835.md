# Cutover L — de-type the `vm` substrate value (resolve-to-envelope)

The `vm:` substrate value is no longer stored typed or consumed by the kernel as
`spec.Vm`. `uf.VM` is opaque; candy/plugin-substrate projects each body into a
`ResolvedVm` via `OpResolve`; the kernel's vm build/deploy consumers read the
envelope, never `spec.Vm`. This is the last of the five substrate-value de-types
(local/android/pod/k8s already landed; I–K), and the last core kind de-type before
the structural gate (N).

## What moved

- `uf.VM` is now `map[string]json.RawMessage` (was `map[string]*VmSpec`), folded via
  `foldOpaqueTemplateReply` + merged via the generic `mergeRawTemplateMap` (typed
  `mergeVmMap` deleted, R5) — consistent with the pod/k8s/local/android substrate maps.
- candy/plugin-substrate gains a `vm` `OpResolve` arm: `spec.Vm` → `ResolvedVm`
  (field-copy + Raw). The 8 kernel chokepoints that read `uf.VM[name]` now resolve via
  `resolveVmViaPlugin` (check_include, check_cmd, bundle_add_cmd, vm_build,
  validate_preempt, vm.go, vm_import, vm_lifecycle_preresolve, k3s_post).
- The `VmSpec` alias is now `spec.ResolvedVm` (one flip in `vmshared/spec_aliases.go`),
  so the remaining 26 consumers + the vmshared render (qemu/cloud-init/libvirt) read
  the envelope unchanged. `LifecyclePrepareInput.VM` is `*ResolvedVm`; the vm deploy
  plugin (candy/plugin-deploy-vm) reads it, and `charlyInstallStrategy` now takes
  `*ResolvedVm`.
- **No charly `*.go` imports `spec.Vm`** — even the pre-decode
  (`decodeTemplateValueJSON[VmSpec]`) now uses `ResolvedVm`, so vm has no host
  pre-decode residual (cleaner than the other four substrates).

# Cutover N — lock the kernel/plugin boundary law (structural gate capstone)

The de-typing program's capstone: the kernel's LAST concrete-kind residual — the substrate
TEMPLATE pre-decode — is removed, and a structural gate asserts the law mechanically so it
cannot regress. With this, no `charly/*.go` production code consumes a concrete-kind
`spec.<Kind>` type; every vocab kind (sidecar/agent/init/resource/distro) and every
substrate value (local/android/pod/k8s/vm) resolves to a `Resolved*` envelope via its
plugin's `OpResolve` (Cutovers D–M).

## What moved

- `decodeStandaloneTemplateJSON` (the substrate-template canonicalization) no longer decodes
  into a typed `spec.<Kind>` via `decodeTemplateValueJSON[XSpec]`. It now canonicalizes
  GENERICALLY through the new shared `entityBodyJSON` (yaml.v3 → `map[string]any` → JSON) —
  the SAME body→wire mechanism `runPluginKind`'s `op.Params` path already used (R3: one
  mechanism, refactored to call it). The value is validated kind-blind
  (`validateKindValueCUE` against the data-driven `#<Kind>Value` def); the plugin `OpLoad`
  re-decodes + defaults + echoes, so no host typed decode is needed.
- Deleted (R5): the now-dead `decodeTemplateValueJSON` generic helper, and the 4 unused
  substrate aliases `PodSpec`/`K8sSpec`/`LocalSpec`/`AndroidSpec` from `vmshared_aliases.go`
  (relocated to a `_test.go` file — only tests build the concrete authored shapes).
  `VmSpec` stays (it is `= spec.ResolvedVm`, an envelope, 35 legit consumers).
- Added `TestNoConcreteKindInKernel` — the structural gate. AST-based (comment-free), it
  fails if any spec-importing production `charly/*.go` references a concrete-kind
  `spec.<Kind>` type (Vm/Pod/K8s/Local/Android/Distro/Sidecar/Agent/Resource). `Init` is
  omitted (the unrelated `ShellSpec.Init` field shares the name; init's de-type is F-proven).
  Verified with teeth: a re-introduced `spec.Vm` trips it; removed, it passes.

This closes the plugin-first de-typing program (Cutovers 1, A–N): the kernel is a
kind-blind execution substrate; every capability is a plugin candy.

# Hard Cutover: SIZE is the same forbidden argument as difficulty (forbidden-excuse catalog)

Strengthen CLAUDE.md "Hard Cutover by Default" — the Forbidden-excuse catalog — to close the
size-based loopholes (motivated by a session where "it's huge / a multi-cutover sub-program /
can't finish this session" was used to defer, and working spike code was reverted citing size):

- Difficulty AND SIZE are both forbidden scope arguments; name the size framings explicitly.
- A large cutover DECOMPOSES into atomic per-unit cutovers, each landed NOW — decomposition is
  forward motion, never a deferral device.
- Reverting / discarding already-working code because the remaining scope is large is the
  WORST form of the excuse.
- The excuse stays forbidden when dressed as "an honest assessment" / "scoping finding" /
  "the spike's deliverable".

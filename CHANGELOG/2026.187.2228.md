# H part 2d — kube resolves its cluster context via the generic reverse-leg (delete the op-rewrite)

Fourth endpoint-verb migration. Unlike cdp/vnc/spice (endpoint verbs), kube shipped NO Substrate
— it had a host-side OP-REWRITE (preresolveKubeCluster) that filled `kube_context` from a
`cluster: <profile>` lookup before marshaling. Now the out-of-process candy/plugin-kube resolves
it itself via the new generic `cc.ResolveClusterContext("<profile>")` reverse-leg.

## What moved

- Host: `preresolveKubeCluster` (the kube-specific op-rewrite in the verbPreresolvers registry)
  → `resolveClusterContext(cluster)` (k8s_config.go), the CheckContext.ResolveClusterContext
  leg for BOTH the in-process runnerCheckContext and the out-of-process checkContextReverseServer.
  It reads the same findK8sSpec project loader; an empty context (no matching profile) is a valid
  result (the plugin falls back to the kubeconfig current-context).
- Plugin: `candy/plugin-kube` calls `cc.ResolveClusterContext(in.Cluster)` when `cluster:` is set
  and `kube_context:` is not, then builds its rest.Config from the resolved context.
- Deleted (R5): `preresolveKubeCluster` + the kube verbPreresolvers entry; the now-unused `kit`
  import in k8s_config.go; stale doc refs swept (mcp / verb_builtins / findK8sSpec). Uniform-API
  gate updated for the new RPC. `findK8sSpec` stays (resolveClusterContext + from-box consume it).

# H part 2e — mcp resolves via the generic reverse-legs; the endpoint-preresolver + Substrate infra retired

The FIFTH and last endpoint-verb migration, completing the check-verb endpoint seam. mcp no
longer has a host preresolver; candy/plugin-mcp resolves its own context via the generic
`cc.ResolveImageLabel("ai.opencharly.mcp_provide")` (the declared servers) + `cc.ResolveEndpoint`
(the host-routable URL). With mcp migrated, ALL five verbs (cdp/vnc/spice/kube/mcp) are off the
old mechanism, so this cutover ALSO retires the entire check-verb preresolver + Substrate infra.

## What moved

- Host: `resolveImageLabel(label)` (check_endpoint_resolve.go) — read one raw OCI label off the
  deployment image (resolveContainer + InspectLabels), for both the in-proc runnerCheckContext
  and the out-of-proc checkContextReverseServer. `MCPProvideEntry` + `podAwareMCPProvides` moved
  to sdk/spec (shared with the plugin, R3); provides.go aliases the type and config_image.go +
  GlobalEnvForImage call spec.PodAwareMCPProvides.
- Plugin: candy/plugin-mcp/resolve.go builds the endpoint from the legs — parse the mcp_provide
  label (spec.CandyMCPProvide) → template {{.ContainerName}} → spec.PodAwareMCPProvides → pick →
  rewrite the picked URL's port via cc.ResolveEndpoint. The relocated pure logic
  (pick/template/URL-rewrite) is unit-tested in the plugin.
- RETIRED (R5): the whole check-verb preresolver mechanism — `verb_preresolve.go`,
  `verb_preresolve_wrappers.go`, the `verbPreresolverFor` dispatch in invokeVerbProvider, and the
  `CheckEnv.Substrate` channel; plus `mcp_preresolve.go` (+ its test). Every host-coupled verb now
  resolves through the class-generic CheckContextService reverse-legs (ResolveEndpoint /
  ResolveGraphicsEndpoint / ResolveClusterContext / ResolveImageLabel). Uniform-API gate updated;
  stale doc refs swept.

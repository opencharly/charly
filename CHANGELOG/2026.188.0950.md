# PR-only agent-validated landing — no direct pushes to main (superproject half)

The runtime-enforcement half of the project-wide cutover to a PR-only,
agent-validated landing model. A **direct push to `main` is forbidden and
mechanically disabled** across every repo; every change lands through a PR that a
FRESH `pr-validator` agent independently validates and merges, generating and
rewriting the merge-time CalVer.

- **`.claude/hooks/pre-push-gate.sh`** — now also blocks any `git push` whose
  refspec destination is `main` / `refs/heads/main` (the `feat/` and `refs/tags/…`
  pushes the flow needs stay allowed; a bare `git push` is left to the
  authoritative server-side branch protection). The force-push / `--no-verify` /
  `core.hooksPath` blocks are unchanged.
- **`scripts/apply-branch-protection.sh`** — idempotent apply/verify helper that,
  per repo, sets rebase-only merge + `delete_branch_on_merge` and PUTs branch
  protection on `main`: require a PR (0 approvals — fully autonomous), require the
  `charly/claude-validation` status (strict), `required_linear_history`, block
  force-push + deletion, `enforce_admins=true`. The protection body is the exact
  shape proven on the policy's RDD spike.
- **`.github/pull_request_template.md`** — the canonical PR template (summary · how
  tested / R10 evidence · attribution tier · R0–R10 checklist · a placeholder
  CHANGELOG the evaluator finalizes at merge).
- **`CLAUDE.md`** — "Post-Execution Policies" Auto-land rewritten to the two-step
  PR flow; the Acceptance-checklist Landing bullet, the hooks-doctrine (now naming
  the main-push block), the agent roster (adds `pr-validator`), and the Key-Rules
  git dispatcher updated.
- **`plugins` pointer** bumped to the merged plugins half (the `pr-validator` agent
  + the git-workflow/agents/migrate/… skill rewrites).

Proven end-to-end by dogfooding: the plugins half landed through this very flow
(branch protection → PR → a fresh `pr-validator` posted `charly/claude-validation`,
finalized the merge-time CalVer, and rebase-merged) — the evaluator even caught a
real incomplete-R5-sweep on its first pass and blocked the merge until fixed.

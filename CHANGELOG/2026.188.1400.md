## Concurrency + correctness fixes surfaced by this cutover's own 61-bed gate
## (each R1-RCA'd from the actual error line; all unit-covered)

- **Retention-untag race** (the fan-out's dominant failure): image-tag
  retention pruned GLOBALLY after every build, untagging FROM pins that
  in-flight sibling builds still resolve (registry-403 fallbacks) and deleting
  last tags mid-build (buildah layer-store corruption → "layer not known" +
  podman SIGSEGV). Fix: build-activity locks (`acquireBuildActivityLock`,
  flocked nonce files recording each live build's generate CalVer) + a prune
  floor (`liveBuildFloor`) — retention never removes a tag at/above the oldest
  live build's CalVer, protects everything when a floor is unreadable, and
  never deletes an image's LAST tag while any build is live
  (`retentionRemovable`, pure + table-tested).
- **`charly migrate` overlay dead-end**: a project already at schema HEAD
  short-circuited BEFORE the per-host overlay leg, leaving a lagging
  `~/.config/charly/charly.yml` in an unresolvable "Run: charly migrate" loop.
  Fix: overlay-aware early exit + per-side (project/host) gating of each
  migration entry (candy/plugin-migrate/engine.go; regression-tested;
  live-verified on this host's floor-version overlay).
- **qcow2 fetch-cache race**: per-cachePath flock + double-checked cache-hit +
  stale-resume invalidation (a RESUMED partial failing verification refetches
  once from zero — mutable `latest` URLs rotate) (http_fetch.go).
- **localpkg shared-dir build race**: per-srcDir flock (defensive mutual
  exclusion of pkg/<fmt>'s shared src/ working copies).
- **Namespaced ensure-image build fallback**: `buildableShortName` now resolves
  a QUALIFIED ref (`fedora.fedora-builder`) directly — a bed racing its
  siblings to first-build a namespaced builder no longer dies on the registry
  403 path.
- **check-preempt-local bed assertion** scoped to bed-owned state (a global
  "no leases" assertion can never hold under a parallel roster).
- **arch-pacstrap `backend: libvirt` pin** (auto silently fell back to qemu
  under load, failing every libvirt-RPC probe; R3 sweep confirmed all other
  probe-backed VM entities already pinned).
- **verify-beds workflow**: discovery now returns `requires_exclusive` tokens;
  same-token beds run as serial chains inside the parallel fan-out.
- nil-deref hardening: `CheckLiveCmd.runVm` no longer discards LoadUnified's
  `ok`. sdk submodule re-attached to `main` (a detached HEAD broke makepkg's
  `git+file` sdk source deterministically — the failure once mis-blamed on
  build concurrency).

# 2026.189.0110 — ai.opencharly.env label reader accepts the object it bakes

`writeLabels` bakes `ai.opencharly.env` as a JSON OBJECT (the image's
`spec.Box.Env` map), but `ExtractMetadata` decoded it straight into
`BoxMetadata.Env` (`[]string`), so every image carrying a box-level `env:` map
failed `charly check box` with "parsing ai.opencharly.env: json: cannot
unmarshal object into Go value of type []string" (surfaced by
`check-android-emulator-pod`; a genesis-era mismatch). Fixed the reader: decode
the object into a `map[string]string`, then convert to the `[]string` KEY=VALUE
form every deploy consumer (`ResolveEnvVars`, the start/shell deployEnv) expects
via the existing deterministic `envMapToPairs` — the exact inverse of the bake,
symmetric with the overlay-merge path (`deploy.go`). Live-verified:
`charly check box android-emulator` exit 0.

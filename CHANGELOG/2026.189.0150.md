# 2026.189.0150 — fail-fast SKIP for GPU beds with no matching host card

A check bed claiming a GPU resource (`requires_exclusive` / `requires_shared`
mapping to a `resource:` gpu vendor) on a host with no matching card used to
build a 10-30 GB image and only THEN die at start with "unresolvable CDI
devices nvidia.com/gpu=all" (the shared-nvidia pods) — or hard-FAIL at
vm-create (the exclusive nvidia VMs). On this AMD-only host that made three
NVIDIA-passthrough beds unrunnable-but-reported-as-FAIL. Added a deterministic
pre-build host-prerequisite check (`bedGPUPrereqMissing` /pure `gpuPrereqMissing`,
reusing `DetectVFIO` + `selectGPUByVendor`): BEFORE any build or resource
acquire, a bed whose required GPU vendor is absent is SKIPPED — a new
`CheckSkippedExitCode` (3) + a "SKIPPED — no GPU matching vendor …" line, like a
missing `/dev/kvm`, never a FAIL. `bedRunResult` gains `SkippedPrereq`/
`SkipReason`; `charly check run` and `main()` map it to exit 3; the verify-beds
workflow reads exit 3 as `skippedPrereq` DETERMINISTICALLY (was agent-inferred).
Live: `check-cachyos-gpu-vm` → SKIPPED exit 3 with no build; `check-local`
(non-GPU) → PASS exit 0 unaffected.

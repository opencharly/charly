# 2026.189.0237 — pgvector moves from AUR to the extra repo (arch)

`pgvector` moved out of the AUR into Arch's official `extra` repo
(`extra/pgvector 0.8.4-1`). Under `candy/postgresql`'s `distro.arch` it was
still declared as an `aur:` package, so on an arch build `yay -S --needed`
resolved it as a REPO package, makepkg never ran, the aur builder's
`find … *.pkg.tar.zst -exec cp` copied nothing, and an EMPTY `/tmp/aur-pkgs`
COPY layer was committed + cached — the `check-cachyos-immich-ml-pod` build
failure ("`/tmp/aur-pkgs/*.pkg.tar.zst`: could not find or read package"). Moved
pgvector to the plain `package:` list (it installs via `pacman` now); it was the
only arch AUR package for postgresql, so the whole aur builder stage is
eliminated from every arch image composing postgresql. The fedora path already
carried `pgvector` as a repo package — this brings arch in line.

Verified: `pgvector 0.8.4-1` is in `extra` (RDD); the regenerated immich-ml
Containerfile installs it via `pacman -Syu … postgresql pgvector` with ZERO aur
stages; `pacman -Sp postgresql pgvector` resolves all three from `extra`; and a
fresh arch immich-ml build executed that step (`installing pgvector…` →
`Detected PostgreSQL 18`).

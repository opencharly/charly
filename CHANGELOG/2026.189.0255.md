# 2026.189.0255 — adopt sdk aur-builder empty-output guard

Bumps the `sdk` pointer to `6c20071` (v0.2026189.249): the `aur` builder stage
now fails loudly when a non-empty `aur:` package list produces ZERO
`.pkg.tar.zst` artifacts — the R8 masked failure that let an empty COPY layer
cache when a package moved from the AUR into a repo (the pgvector incident's
prevention half). No superproject code change — a submodule-pointer adoption of
an already-validated sdk fix.

Behavioral gate (this consumer PR, per SDK-first landing): a fresh
`charly box build arch-test` (arch, `arch-aur-test` installs `yay-bin` — a
GENUINE AUR package) built EXIT 0 with the guard present (`installing
yay-bin…` → `Successfully tagged arch-test`), proving the guard does NOT
spuriously fail a legitimate aur build. The zero-output failure path is proven
by the sdk-side shell-logic test (exit 1 on empty).

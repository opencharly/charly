# 2026.189.0410 — generate hard-errors on a localpkg render failure (R8 masked failure)

`writeCandySteps` demoted a `renderLocalPkgImageInstall` ERROR to a Containerfile
COMMENT (`# localpkg render error: …`) and kept building — masking the failure.
A `--dev-local-pkg` build whose charly-toolchain makepkg failed therefore shipped
an image with NO `/usr/bin/charly`, only surfacing DOWNSTREAM as failing charly
checks (the `check-charly-selftest-pod` defect: 5 checks failed because the
binary was absent). This directly contradicted `renderLocalPkgImageInstall`'s
own contract ("must fail loudly, never silently fall back"). Now the render
error is returned (hard-fails the generate/build) instead of a swallowed comment.

Verified: a fresh `charly box build charly-selftest --dev-local-pkg` — where the
render SUCCEEDS — builds EXIT 0 with `opencharly-git` installed and ZERO
`localpkg render error` comments in the emitted Containerfile (the changed
generate path ran live, proving no regression on the success path); the
error-branch is a direct `return err` and the old masked-comment path is
grep-verified gone (R5).

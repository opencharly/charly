# 2026.189.0450 — PR template elicits full per-rule + R10 evidence for the validator

Expands the PR template so an author supplies EVERYTHING the pr-validator needs
to verify CLAUDE.md compliance — the validator can only enforce what the body
provides. The "How tested (R10)" section becomes "How R10-tested — exact gate,
commands, and pasted output": it now prompts the exact change-class gate, the
`disposable: true` target name, the fresh-rebuild/R9 confirmation, whether the
CHANGED code path (not just the success path) executed live (which caps the
tier), and — for shared-state changes — the CONCURRENT roster evidence with the
"no idle/serial passes" rule stated inline. The rule-compliance checklist grows
from a partial R1–R9 tick-list into a full R0–R10 + pillars section where each
line asks HOW the rule is satisfied (or `N/A — <reason>`), one prompt per
pr-validator checklist item: R0 skills, R1 RCA+zero-warnings, R2, R3, R4, R5
grep, R6, R7, R8, R9, R10 coverage, RDD/ADE/SDD, hard-cutover, kernel/plugin
boundary law, disposable-only autonomy, clean-arch+Go-gates, CHANGELOG. The
header states a bare checkbox / blank section / promise FAILS.

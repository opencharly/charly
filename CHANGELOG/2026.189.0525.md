# 2026.189.0525 — remove per-repo PR template; inherit the org .github default

Deletes the superproject's `.github/pull_request_template.md` so it falls through
to the single org-wide source `opencharly/.github` (bootstrapped this session with
the canonical rich template). Part of the R3-at-the-repo-level consolidation: the
same template previously lived duplicated in 10+ repos — editing it meant editing
every copy. GitHub serves a repo named `.github` at the org level as the DEFAULT
community-health source for every repo without its own, so one file is now the
single source.

The AUTHORITY for what a PR must contain remains CLAUDE.md + the
`/charly-internals:git-workflow` + `pr-validator` skills (the 0–18 checklist); the
template is the GitHub-UI mirror of that, now single-sourced.

# 2026.189.1230 — decouple ResolvedBox.InitConfig → Generator (P3-unit-3)

Removes the transient `InitConfig *InitConfig` field from `ResolvedBox` (a
`json:"-"` resolution helper) and carries the one project init: vocabulary on
the `Generator` instead. Init-system resolution (`ActiveInit`/`ResolveInitSystem`)
runs over `Candies` + candyOrder — a build/deploy-time concern, not per-image
config — so it belongs on the Generator, not each box. `NewGenerator` sets it
from the `defaultInitCfg` it already loads; the four consult sites read
`g.InitConfig` (generate.go) / `t.Generator.InitConfig` (deploy_target_pod.go).

This unblocks the `ResolvedBox` → sdk/buildkit move (P3): with the InitConfig
field gone, `ResolvedBox`'s remaining field types are all sdk-resolvable
(`DistroConfig`/`BuilderConfig` in buildkit, `MergeConfig`/`DistroDef`/
`ResolvedInit` in spec). Behavior-neutral; golden tests + check-builder-local +
check-local beds green.

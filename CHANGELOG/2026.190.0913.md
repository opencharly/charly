# 2026.190.0913 — deploy/IR machinery → sdk (deploykit + kit + spec + sshx); the CandyModel keystone (P4)

Moves the whole deploy/IR machinery out of charly core into the sdk, so an
out-of-tree plugin can compile deploy plans, construct executors, walk trees, and
read/write the deploy ledger — the literal form of "a plugin can do EVERYTHING"
(kernel/plugin boundary law: the kernel keeps generic Mechanisms; the machinery
becomes importable libraries). charly core now consumes it through thin alias
re-exports (`charly/deploykit_*_aliases.go`, `charly/kit_*_aliases.go`) — every
call site compiles unchanged.

What moved:
- **`sdk/deploykit`** — the deploy-plan compiler (`install_build.go` —
  `BuildDeployPlan` + the `compile*` helpers, `HostContext`, `MergePlan`,
  `DescribePlan`, `ComputeDeployID`), the step vocabulary + `step_view.go`, the
  tree walk + executor pick (`deploy_tree.go`/`deploy_chain.go`), the deploy-state
  model (`deploy_state.go`), `plan.go`, `provides.go`, `bundle_derive.go`.
- **`sdk/kit`** — executor implementations (`deploy_executor*.go`,
  `deploy_host_shell.go`, `deploy_types.go` — ShellExecutor/SSHExecutor/Nested),
  `install_ledger.go`, `reverse_ops.go`, `builder_run.go`, the `runtime_config.go`
  store, and the pure-helper batch (`env`/`envfile`/`http_fetch`/`ports`/
  `transfer`/`engine`/`container_name`/`readiness`).
- **`sdk/spec`** — the IR E-envelope (`install_plan.go`, `deploy_executor.go`).
- **`sdk/sshx`** — `ssh_client.go` + `ssh_tunnel.go`, re-homed from `vmshared`.

**The CandyModel keystone.** The compiler reads a candy through the new
`deploykit.CandyModel` INTERFACE, not the concrete runtime `Candy` — so `Candy`
and its ENTIRE test suite STAY in charly, which implements the interface via
`charly/candy_model_accessors.go` (`GetName`/`GetSourceDir`/`GetVersion`/`Vars`/
`PlanSteps`/`Reboot`/`HasFile`). Zero test-fixture churn — no `Candy{…}` literal
was touched. The charly-coupled compiler legs stay charly and inject as SEAMS
(`deploykit/compile_seams.go`): `CompileActOp` (provider registry),
`CompileServiceSteps` (loader + `RenderService`), `OpInContext` (VerbCatalog).
`ResolvedBox` reaches deploykit from `sdk/buildkit` (acyclic).

Hard cutover: the old `charly/{install_build,deploy_chain,deploy_executor*,env,
envfile,http_fetch,install_ledger,reverse_ops,runtime_config,step_view,…}.go`
are DELETED in the same change — `git grep` finds the moved identifiers only via
the alias re-exports (their intended end state) + this `CHANGELOG/`. The move
orphaned nine unused alias re-exports (`externalStepKindPrefix`, `stepsToView`,
`argHasImageTag`, `appendOrReplaceEnv`, `deriveChildExecutor`,
`appendHopForFlatPath`, `resolveAutoEnable`, `detectEngine`) and the dead
`stripManagedBlock` (its behavior moved to `kit.RemoveManagedBlockAt`) — all
deleted, with their stale doc-comments swept to the surviving `deploykit`/`kit`
symbols (R5).

Behavior-neutral: `go test ./...` green both repos; gofmt clean;
`golangci-lint run ./...` → 0 issues both repos; the disposable check roster
green (local/pod/vm + builder/exampledeploy/substrate/k8s/group/structkind/
commands/agent/sidecar/migrate/feature/doctor/udev/preempt), fresh-`charly
update` gate included.

# 2026.190.1223 ŌĆö pre-commit / pre-push gates: simplify to a real backstop, deduplicate, fix the `-C` fail-open

The `pre-commit-gate` detected `git commit` invocations with a regex that could not
span a quoted argument containing a space. When the match failed it did not complain ŌĆö
it concluded there was no commit and allowed everything, so `git -C "$P" commit` (a
variable) blocked with a misleading message and `git -C "/has space" commit` bypassed
the gate outright. The fix for that grew, across several iterations, into a 200-line
shell-parsing engine ŌĆö line-continuation normalization, `eval`/`bash -c` recursion, a
command-wrapper safety net ŌĆö chasing obfuscations (`g''it`, `base64 | bash`) that a
static gate provably cannot close and that no honest commit ever uses.

That was the wrong shape. These gates are **discipline backstops, not security
boundaries**: every change is independently re-validated by a fresh `pr-validator`
agent and by GitHub server-side branch protection. So they are rewritten to catch the
common, honest mistakes cheaply and delegate nuance and the adversarial case to the
agent and the server.

**What changed.**

- **One shared helper, `.claude/hooks/gitcmd.py`** ŌĆö `git_invocations()` finds
  `git [globals] <subcommand>` with a plain `shlex.split` (quotes handled, so a quoted
  path with a space is one token ŌĆö the one real fix), plus `hooks_path_override()` and
  `dash_c_dir()`. Both gates use it; the parsing that was duplicated across the two
  hooks (and had the same fail-open in each) now lives once (R3).
- **`pre-commit-gate.sh`** enforces exactly its documented invariants, each an
  independent best-effort check so no single fragile parse can disable the rest:
  hook-bypass (`--no-verify`/`-n`/`core.hooksPath`), a missing `Assisted-by:` trailer,
  an illegal tier, `documentation reviewed` on a non-docs diff, and a runtime tier with
  no staged CHANGELOG entry. A `-C <dir>` that does not resolve to a real directory
  (a `$VAR`, a nonexistent path) makes the diff-dependent checks **skip** rather than
  inspect the wrong repo ŌĆö the `pr-validator` still covers tier-vs-diff.
- **`pre-push-gate.sh`** enforces force-push, `--no-verify`/`core.hooksPath`, and
  direct-push-to-`main` over the same shared parser ŌĆö which also closes the identical
  space-in-a-quoted-arg fail-open it carried.
- **Deleted**: `_normalize_shell`, the `eval`/`bash -c` recursion, the wrapper safety
  net, `resolve_literal_dir`'s expansion analysis, the segment tokenizer ŌĆö the whole
  anti-obfuscation layer. Obfuscation is out of a local gate's reach by construction
  (splitting the command word or generating it at runtime defeats any static parser in
  an infinite regress); the authority for the adversarial case is GitHub branch
  protection (`enforce_admins` + the required `charly/claude-validation` status).

**Verified** by a 27-case behavioural suite on real throwaway repos: every invariant
blocks its violation and allows its honest form ŌĆö including the original bug
(`git -C "$P" commit` no longer misfires; `git -C "/has space" commit` at docs tier
correctly inspects that repo), `-C` to a submodule, `--no-verify` inside a message not
tripping, commit's own `-c HEAD~1` not mistaken for a global, `echo "git commit"` not
treated as a commit, and the full push-gate set.

## What P5 actually is, versus what the plan said

The approved plan describes P5 as relocating ~25 files to `sdk/kit` behind "the ONE named
blocker: verb resolution". That is not executable as written, and the operator approved an
amended HOW (end-state unchanged, seam rewritten). Verified against the source, not assumed:

- **`Runner` cannot move.** It is defined in `charly/checkrun.go`, but EIGHT of its methods are
  declared in files the plan keeps in core (`provider_checkenv.go:78,136`, `k8s_config.go:62`,
  `checkrun_charly_verbs.go:33`, `check_endpoint_resolve.go:25,61,164,197`). Go requires all
  methods of a type to live in the type's package. The engine must be **rewritten against
  `kit.CheckContext`**, not relocated — `Runner` stays in core as the concrete implementation
  (the adapter `charly/check_kit_adapter.go` already maps every method onto it).
- **`charly/description_include.go` does not exist** — the plan names a phantom file. The real
  one is `charly/check_include.go`.
- **`checkspec.go:420` type-asserts `*grpcProvider`**, a concrete transport type that cannot
  follow into the sdk. The "is this out-of-process?" question must become an sdk-level concept.
- **`run_subcommand.go` re-execs the charly binary** and is a reverse dependency of four core
  deploy files. It stays in core, contra the plan's move list.

A `k8sgen_wire.go` precedent also settles a question the plan left open: the host lifts what a
plugin needs into a narrow envelope "so the plugin needs no access to the package-main
`BoxMetadata` type". The moved resolver will take the same projection, so the unfinished
P2 unit (B) is **not** a P5 prerequisite.

## Two bugs fixed upstream

Writing the tests the 298-line parser never had (it had **zero** coverage) surfaced two real
defects, both fixed in `sdk` `v0.2026191.1413`: the lexer decoded bytes as runes, so a non-ASCII
tag was unusable (`ParseTagExpr("café")` → `unexpected character '©' at position 4`), and invalid
UTF-8 lexed silently instead of erroring.

A first version of that change also collapsed the leading-character and continuation identifier
predicates, silently widening the grammar; a `pr-validator` round caught it, and the sdk CHANGELOG
records the correction. Behaviour parity against the verbatim original is proven by a 36-input
differential — identical on every ASCII input.

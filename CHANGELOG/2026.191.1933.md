# 2026.191.1933 — core consumes the kit result model + plan-execution helpers (P5 units 2+3 consumer)

Consumer half of `sdk` `v0.2026191.1732` (the check **result model**) and `v0.2026191.1837`
(the **plan-execution helpers**) — landed as ONE superproject cutover.

Deleted from core, now aliased to `sdk/kit` through `charly/kit_aliases.go`:

- **Result model (unit 2):** `charly/description_report.go` (the text/JSON/TAP/JUnit formatters);
  the `CheckStatus`/`CheckResult`/`StepResult` struct definitions; and the `kitStatusToCheck`
  converter. `CheckStatus` is now a type alias of `kit.Status` (the two were the same
  three-valued enum bridged by that converter), so a verb's `kit.Result` verdict flows into a
  `CheckResult` with no conversion.
- **Plan-execution helpers (unit 3):** `charly/description_eventually.go` (the `eventually:`
  retry loop), `charly/description_capture.go` (`ScenarioContext`), `charly/check_level.go` (the
  acceptance-depth ladder) — with their tests. `plan_validate.go` and `check_substitute.go` stay
  in core (dual-invoked by box-validate + the feature HostBuild seam; and bound to the core CUE
  loader). The `ExtractMetadata` label round-trip stayed too — split to
  `charly/check_level_labels_test.go`.

Dead aliases the moves left behind were removed: `plural`, `signalKillErrMarker`
(lowercase — `golangci-lint`'s `unused` flagged them), and `IsValidCheckLevel`, `CheckLevels`,
`CheckLevelNone` (exported — `unused` cannot see a caller-less exported alias in `package main`,
so an exhaustive `git grep` per alias is the only check that catches them). Core aliases only
what core actually calls; every surviving alias in `kit_aliases.go` has a verified core caller.

The grammar/result model now lives ONCE in kit, so a plugin candy that runs a plan shares the
retry loop, capture context, acceptance ladder, result types, and reporters with the check
engine instead of re-implementing them.

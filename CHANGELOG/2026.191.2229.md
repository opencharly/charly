# 2026.191.2229 — core consumes the check-engine plan walk in sdk/kit (P5)

Consumer half of the P5 check-engine SDK-ization. The check engine's plan walk
moves to `sdk/kit`; core keeps only the host-serving atoms and drives the walk
through the new kind-blind seams.

## The plan walk dissolves into kit

`RunOne` (checkrun.go) + `RunPlan`/`runUnit` (description_run.go) moved to
`sdk/kit` (planrun.go). Core now:

- `Runner.Run` calls `kit.RunOne` per check; `RunPlan` delegates to `kit.RunPlan`.
- `charly/planrun_adapter.go` adapts the live `*Runner` to `kit.PlanContext` +
  `kit.VerbResolver` — wrapper structs (not methods on `*Runner`, which already
  has fields named `Distros`/`Mode`/`VerifyOnly`/…, the same reason
  `runnerCheckContext` wraps). `runnerVerbResolver.RunVerb` is the former
  `providerRegistry.ResolveVerb` + `CheckVerbProvider`/`invokeVerbProvider`
  dispatch; `runnerPlanContext.ContextSkipReason`/`SwapVenue`/`EffectiveDo` wrap
  the grammar + venue swap the inline `runOne` did.

The `VerbCatalog` grammar (`opEffectiveDo`/`opEffectiveContexts`/`opInContext`/
`opActsInBuildDeploy`) stays in `checkspec.go` — it consults the provider
registry and `ExecContext` (which lives in deploykit; kit can't import it) — and
is reached through the `PlanContext` seam. `ExecContext` never crosses the kit
interface.

Moved out of core to kit (aliased in `charly/kit_aliases.go`): the `DoMode` enum
(checkspec.go), `LabeledDescription`/`LabelDescriptionSet`+`IsEmpty` (labelset.go),
`GraderRequest`/`StepGrader` (check_feature_grader.go — `AgentGrader` stays as the
production impl), and the pure step helpers `KeywordOf`/`StepDoMode`/
`EffectiveStepID`/`FilterHostVars` (description_spec.go / check_members.go). The
now-dead `runWithEventually` alias was removed.

## The ${NAME[:arg]} expansion consumer (folded in)

Also the consumer of sdk `v0.2026191.2033` (the check-variable expansion grammar,
which landed in `sdk/kit/checkvars_expand.go`): the 219-line expansion cluster is
deleted from `charly/checkspec.go` and aliased to `sdk/kit`
(`TestVarRefs`/`IsRuntimeOnlyVar`/`opExpandVars`/`collectAnyStrings`).
`ExpandTestVars` is NOT aliased — core production expands via `opExpandVars`
(which wraps it internally) and the sole other caller, `checkvars_test.go`, calls
`kit.ExpandTestVars` directly, so a package-main alias would export a name no
production code calls.

## What stays core (moves later)

The loader-coupled check files (`readiness_config`, `check_score_kind`,
`check_members` resolution) move with the loader in P6; the harness-coupled
`check_substitute` moves with the AI harness in P12; the `VerbCatalog` grammar +
`opActsInBuildDeploy` (registry-coupled) stay core behind the seam. Only the
vocabulary-independent walk moved.

# 2026.192.0835 — the loader parse becomes a swappable plugin + the typed uf.* maps die (P6 consumer)

Consumer half of P6 (loader → plugin), ONE cutover with two coupled halves: (1) the node-form config
PARSE now runs in `sdk/loaderkit`, served by a compiled-in loader plugin candy, and core DELETES its
own parse; (2) the last typed per-kind loader maps — `uf.Box` (`map[string]BoxConfig`) and `uf.Candy`
(`map[string]*InlineCandy`) — fold to the generic kind-keyed `map[string]json.RawMessage`, closing the
loader incomplete seam so the kernel holds no per-kind Go map.

## The loader plugin is the config front-end

- **`candy/plugin-loader`** — a compiled-in plugin candy implementing `loaderkit.DocParser` (the
  default node-form parse via `sdk/loaderkit`). Registered at init() before the first load — no
  bootstrap cycle (the registry seed reads the embedded `providers:` manifest via a plain
  `yaml.Unmarshal`, never `LoadUnified`). Listed in `charly.yml` `compiled_plugins:`.
- **`ClassLoader`** (`charly/provider.go`) — the new `loader` provider class; `#PluginCapability`
  admits `loader:loader` (sdk `schema/candy.cue`).
- **`activeLoaderParser`** (`charly/loader_threaded.go`) — the registered config front-end,
  wired at registration when a provider implements `loaderkit.DocParser`
  (`plugin_inproc.go`); swapping the loader plugin swaps the front-end. `loaderThreaded()`
  snapshots the registry-derived kind-recognition DATA the parse consults.
- **`mergeUnifiedDocs`** dispatches each document to `activeLoaderParser.ParseDoc` → the generic
  `spec.ParsedProject` → `materializeProject` (`node_parsed.go`: `parsedNodeToGeneric` +
  `normalizeNodeInto`) folds it into the `*UnifiedFile`. The three genericNode consumers
  (candy discovery, `validateNodeFormSteps`, the candy factory) reconstruct from the same parse.

## Core's parse is DELETED (R5)

`parseNode` / `parseNodeTree` / `classifyKind` (`node_parse.go`) and `desugarEntityPlan` /
`desugarStep` (`node_desugar.go`) — plus the now-orphaned `stepKeywordSet` and `genericNode.raw`
field — are removed; only the `genericNode` TYPE stays (the materialize reconstruction target).
The test helpers build genericNodes via the production loader path (`genericNodesFromDoc`), so no
second decomposition survives.

## The typed uf.* per-kind maps die (loader incomplete seam closed)

- **`uf.Box` / `Config.Box` → `boxMap` (`map[string]json.RawMessage`)** — the loader stores a marshaled
  `BoxConfig` per image name; consumers decode on demand via the accessors in
  `charly/uf_box_generic.go` (`BoxConfig(name)`, `HasBox`, `BoxNames`, `SetBox`, `allBoxNames`,
  `eachBox` — the decode-on-iterate view). `foldCandyKind` writes `uf.SetBox`; the kernel holds no
  `spec.Box`-typed map.
- **`uf.Candy` → `candyMap` (`map[string]json.RawMessage`)** — the loader stores a marshaled
  `InlineCandy` per layer name; `ProjectCandies` decodes on demand via
  `charly/uf_candy_generic.go` (`decodeInlineCandy` / `encodeInlineCandy` / `uf.SetCandy`). The
  discovery-manifest field is exported (`InlineCandy.Manifest`, `json:"__manifest"`, `yaml:"-"`) so it
  survives the opaque round-trip; the former unexported `manifest` is gone.
- The already-generic substrate maps (`uf.Pod`/`VM`/`K8s`/`Local`/`Android` = `json.RawMessage`) were
  the shape this folds Box + Candy into — every per-kind loader map is now uniform.
- **R3:** the twin `mergeBoxMap` / `mergeCandyMap` merges collapse into the existing
  `mergeRawTemplateMap` (byte-identical add-if-not-exists on `map[string]json.RawMessage`); both are
  deleted.
- **~30 core consumers** (generate/validate/refs/security/engine/namespace/alias/deploy/…) switch to
  the decode-first accessors; test literals fold via the `boxMapOf` constructor (the test-side analog
  of `encodeBox`). The byte-equivalence tests (`TestSubstrateKind_BothShapesByteEquivalent`,
  `TestCandyKind_BothShapesByteEquivalent`, `plugin_candy_kind_e2e`) prove the fold neutral.

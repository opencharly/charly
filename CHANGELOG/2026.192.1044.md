# 2026.192.1044 — the remote-repo fetch backend becomes candy/plugin-refs (P7 consumer)

Consumer half of P7 (the remote-repo FETCH BACKEND → plugin). Core DELETES its `refs_git.go`, composes
the git primitives from `sdk/kit`, and dispatches every cache-miss remote-candy download through the
compiled-in `candy/plugin-refs` — the swappable fetch backend that unlocks alternative ref backends
(OCI/S3-hosted candies). Behaviour-neutral: the default backend is the same git fetch.

- **`candy/plugin-refs`** — a compiled-in plugin candy (Class `refs`, word `refs`) implementing
  `kit.RefsDownloader` via `kit.DefaultDownloader` (git). In `compiled_plugins:`. Registered at init()
  before the first load — no bootstrap cycle (the compiled-in `command:migrate` already invokes at the
  same deep-in-load point).
- **`ClassRefs`** (`charly/provider.go`) — the new `refs` provider class.
- **`activeRefsDownloader`** (`charly/refs_threaded.go`) — the registered fetch backend, wired at
  registration when a provider implements `kit.RefsDownloader` (`charly/plugin_inproc.go`); swapping the
  refs plugin swaps the backend (git → OCI/S3). Mirrors `activeLoaderParser` (P6).
- **`EnsureRepoDownloaded`** (`charly/refs.go`) — the cache-miss `DownloadRepo(...)` call becomes
  `activeRefsDownloader.Download(...)`. The fetch ORCHESTRATION stays core (local-override resolution,
  cache-hit short-circuit, the post-fetch schema auto-migration via `command:migrate`).
- **`charly/kit_refs_aliases.go`** — core binds the moved kit primitives it still calls DIRECTLY
  (`RepoGitURL`/`GitClone`/`GitDefaultBranch`/`GitLatestTag`/`compareSemver`/`RepoCacheDir`/`RepoCachePath`/
  `IsRepoCached` — `reconcile`, the collection walk, `check_bed_run`, `scaffold`); the `DownloadRepo` alias
  is gone (core dispatches through the plugin).
- **R5:** `charly/refs_git.go` + `refs_git_test.go` DELETED; the moved tests live in `sdk/kit`;
  `charly/debug_notice_test.go` keeps the core `printDebugRetentionNotice` test.

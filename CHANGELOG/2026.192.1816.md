# 2026.192.1816 — the build-mode render engine moves to sdk/deploykit; charly keeps the host-side ENGINE (P8)

The superproject half of P8 (plugin-build becomes the REAL engine). The build-mode Containerfile
RENDER MECHANISM relocates from `charly/` into `sdk/deploykit` (see the sdk half, opencharly/sdk); charly's
render methods become thin delegating shims over a cached `Generator.toDeploykit()`, and the host-side build
ENGINE stays core as the Mechanism reached via `HostBuild("image")`. Behaviour-neutral: `charly box generate`
is BYTE-IDENTICAL across every image (verified on the main + box/fedora image set, incl. the builder,
localpkg, and supervisord-assembly paths).

- **`charly/generate.go` + `charly/tasks.go` + `charly/init_config.go`** — the reusable render primitives
  (`writeCandySteps`, `emitInitAssembly`/`emitInitFragmentStages`/`generateInitFragments`, `emitTasks`, the
  scratch/extract stages, base bootstrap, per-candy env/expose, traefik routes, effective-version/graph/
  order) become one-line shims delegating to `deploykit.Generator`. `toDeploykit()` constructs the render
  Generator and wires the host-coupled SEAMS from core mechanisms: `EmitPluginOp` (Provider registry),
  `CollectBoxPorts`/`ValidateEgress`, `RenderService` (candy/plugin-init), `RewriteHeaderCopyForRemote` +
  `RenderLocalPkgImageInstall` (host-fs), `ResolveInlineBuilder` + `ExternalizedBuilders` (builder-emit
  registry). `charly/graph.go` DELETED (→ deploykit); `charly/graph_shim.go` keeps the CycleError alias +
  the candyModelMap adapter + the delegating wrappers.
- **`charly/candy_model_render_accessors.go`** — the `*Candy` accessors satisfying the grown
  `deploykit.CandyModel` render/init/writeCandySteps surface (`RelayPorts`/`RunOps`/… over the field-backed
  members, `Get*`-named to avoid field/method collisions).
- **STAYS CORE — the host-side build ENGINE (the Mechanism reached via HostBuild):** `Generate` +
  `generateContainerfile` + `emitPreMainCandyStages` + `generateDataImageContainerfile` (orchestration); the
  builder-emit registry cluster (`emitBuilderStages`/`resolveDetectionBuilder`/`emitExternalBuilder*`/
  `resolveBuilderStage`/`builderResolveInputFrom`); `writeLabels` (kind-BLIND Config-metadata — the OCI
  capabilities contract, no per-kind switch); and the host-fs/registry seams
  (`resolveUserContext`/`createRemoteCandyCopies`/`materializeBuildConfigAsset`/`candyByName`). The delegating
  shims are the PERMANENT facade (core engine → deploykit render library), not a transitional dual-mode path.
- **`charly/local_image.go` + `charly/build_stage_atomic.go` + `charly/intermediates.go` +
  `charly/effective_version.go`** — bind onto the relocated `kit.AtomicWriteFile`/`kit.CompareCalVer` +
  `deploykit` effective-version/candy-collect helpers.
- **`charly/generate_env_escape_test.go` DELETED** — moved verbatim to `sdk/deploykit/candy_env_test.go` with
  the `escapeContainerfileEnvValue` helper it covers; the orphaned `escapeContainerfileEnvValue` docstring left
  in `generate.go` is removed, and every comment/test naming a moved identifier (`taskVarRefPattern`,
  `resolveCascadePackages`, `compileShellSnippetSteps`, `writeBootstrap`) is swept to its `deploykit.` home
  (R5 grep self-test clean).
- **`sdk` submodule pointer** bumped to the merged P8 sdk commit; **`charly/go.mod`** sdk require bumped to
  the merged sdk tag (in-tree resolution rides `replace => ../sdk`; the require pins out-of-tree consumers).

# 2026.192.2004 — deploy substrate resolution reads plugin-declared traits, not the kind word (P9 consumer)

The superproject half of P9 (traits/descent de-branching). Every deploy/check CONSULT site that branched on
HOW a substrate behaves (which venue, whether it is a machine, whether it is a chain leaf) now reads the
plugin-declared `#DeployTraits` stamped onto `node.Descent` (the loader-DERIVED descent descriptor), never a
`switch`/`==` on the concrete substrate kind word — closing the boundary-law incomplete-seams. The DECLARED
trait table lives in the substrate plugin; the kernel consults it BY TRAIT.

- **`candy/plugin-substrate/plugin.go`**: declares the canonical per-word `DeployTraits` table — pod =
  container+image_backed+image_context; vm = ssh+machine_venue+exclusive_venue; local = shell+machine_venue;
  k8s = shell+image_context+leaf_only; android = parent. This is the SINGLE source the kernel stamps from.
- **`charly/plugin_provider_common.go` + `charly/install_plan.go`**: `capMeta.traits` +
  `declaredDeployTraits()` carrier twin (mirrors Structural/Validates); `deployTraitsCarrier`.
- **`charly/deploy_tree.go`**: `deployTraitsFor(word)` (registry → declared traits), `effectiveTarget(node)`
  (""→pod), `nodeTraits(node)`, `deployTraitDescent(word)`; `stampBundleDescents` stamps via `kit.StampDescent`.
- **Consult sites de-branched** (now read the trait, not `node.Target == "<word>"`): `bundle_add_cmd.go`
  (deriveChildExecutorForPath → trait transport), `bundle_members.go` (isPod/isVmMember → venue),
  `check_venue.go` / `check_cmd.go` / `check_score_kind.go` (vm/local/parent checks → venue/host_rooted),
  `ephemeral_lifecycle.go` (vm snapshot refcount → venue==ssh), `install_build_services.go` (machine-venue
  bool), `preempt.go` (holder stop selection → trait, keyed on the declared venue). `HostContext.Target`
  (never "vm" — a dead arm) is retired in favour of `MachineVenue bool`.
- **`charly/no_substrate_word_switch_gate_test.go`** (new): the P9 structural gate — an AST scan over BOTH
  `charly/*.go` AND `sdk/deploykit/*.go` that FAILS on any regrown `.Target (==|!=) "<substrate>"`
  behaviour-branch, allowlisting only the loader/classifier (word-PRODUCING Data), the validators, the status
  collectors (P14), and the deploykit `deploy_state.go` findVmDeploy (persisted state where Descent is
  stripped — a tracked persist-descent follow-up). Teeth verified on both trees.
- **External DEPLOY-class substrates resolve to external-in-place traits.** `deployTraitsFor` uses
  `providerRegistry.ResolveKind`, which only resolves KIND-class words — an EXTERNAL `deploy:<word>`
  substrate (e.g. `exampledeploy`) is served by a deploy-target plugin, so it returned nil traits →
  `Venue==""` → `checkLocalTarget` false → `check live` hit "container not running" for a deploy that builds
  no container. Fixed: `deployTraitsFor` returns the external-in-place `Venue: "none"` trait for a recognized
  external deploy substrate (`isExternalDeploySubstrate`, a registry-recognition D-item — not a kind-word
  switch), and `checkLocalTarget` host-routes the `none` venue alongside `shell`/`parent`. Restores the
  behaviour the retired `isExternalDeploySubstrate` guard gave, now BY TRAIT. Proven on the
  `check-exampledeploy` disposable bed (the external-deploy-lifecycle bed).
- **`sdk` submodule pointer** bumped to the merged P9 sdk commit; **`charly/go.mod`** sdk require bumped.

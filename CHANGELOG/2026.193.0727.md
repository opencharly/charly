# 2026.193.0727 — golangci-lint is now a pre-commit criterion (cutover B)

`pre-commit-gate.sh` gains a **Go-lint criterion**: a `git commit` that stages a `*.go` change
whose touched MODULE is not `golangci-lint`-clean is BLOCKED (exit 2). This closes the gap the P10
VM-CLI cutover exposed — its 21 orphaned dead symbols slipped through because the prior gate ran
`go test` + the F11 gate but never the linter; `golangci-lint`'s `unused` is exactly what catches
that class, so making it a mechanical pre-commit backstop means dead/unused code cannot land at all.

- **`.claude/hooks/pre-commit-gate.sh`** — new `assert_go_lint(repo)` + `go_modules_touched(repo)`:
  determines each Go MODULE (a dir with `go.mod`) that has a `*.go` file in the commit (staged, plus
  the late-staged `-a`/`-i`/`-o` set), and runs the CONFIGURED `golangci-lint run` in each (never
  `--fix` — it corrupts the tree — never `--enable-only` — it breaks the cross-linter dedup),
  `GOWORK=off` for `candy/plugin-*` candies (they lint standalone as they build). Whole-MODULE
  analysis because `unused` needs whole-package. **Fail-OPEN** when `golangci-lint` is absent or
  times out (180s) — the `pr-validator` remains the real gate; the criterion is a cheap discipline
  backstop, not a hard dependency. Tier-independent (runs before the tier checks); a docs/CHANGELOG
  commit stages no `*.go` so `go_modules_touched` is empty and the linter never runs.
- **`.claude/hooks/gate_test.py`** — 3 new cases (65 total, 0 failures): a Go module with an unused
  symbol → BLOCK; a lint-clean Go module → ALLOW; a docs-only change in a Go repo → ALLOW (no `*.go`
  staged, linter skipped). The BLOCK/ALLOW cases guard on `golangci-lint` being installed (the gate
  fails-OPEN without it); the docs-only case is tool-free.
- **`CLAUDE.md`** — the "Hooks doctrine" PreToolUse-gates enumeration gains the criterion.
- **`plugins` gitlink** — bumped to the doctrine half (agents skill + go-quality skill; opencharly/plugins).

Verification: `python3 .claude/hooks/gate_test.py` → 65 cases, 0 failures; `bash -n` clean; live
throwaway-repo tests confirmed BLOCK on `func dead(){}` unused, ALLOW on a clean change, ALLOW on a
docs-only commit. The gate's own commits (this cutover) stage no `*.go`, so the new criterion does
not self-block.

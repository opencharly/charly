# 2026.193.1230 — scaffold ENGINE consumed from sdk/kit (P15 sub-unit, superproject half)

The core rewire paired with sdk `feat(kit): relocate the scaffold ENGINE (box new) into sdk/kit`:
the `charly box new` project/candy/box authoring engine now lives in `sdk/kit`, so charly core
CONSUMES it instead of owning it — one shared library for both the core CLI (now) and the Wave-2
`command:box` plugin (next). The `charly box new` CLI surface + behaviour are UNCHANGED.

## What moved

- **`charly/scaffold.go` DELETED** — its sole function `ScaffoldCandy` is now `kit.ScaffoldCandy`.
- **`charly/scaffold_project.go` slimmed** to the authoring-EDIT helpers only (`AddCandyToBox`,
  `RemoveCandyFromBox` + the private `yaml.Node` resolvers). The create-side engine
  (`ScaffoldProject`, `AddBox`) + the seed templates + `saveYAMLNodeFile` moved to `sdk/kit`; the two
  remaining callers now use the exported `kit.SaveYAMLNodeFile` (ONE marshal-to-file home, R3).

## Core rewire

- `NewCandyCmd.Run` → `kit.ScaffoldCandy(dir, name, ComputeCalVer())` — `ComputeCalVer` STAYS core
  (the wall-clock read) and is passed in so kit is clock-free; the handler now owns the
  "Created candy at…" presentation (kit does the fs work only).
- `NewProjectCmd.Run` → `kit.ScaffoldProject`; `NewBoxCmd.Run` → `kit.AddBox`.
- Scaffold tests split: the engine tests (`ScaffoldCandy`/`ScaffoldProject`/`AddBox`) live in
  `sdk/kit/scaffold_test.go`; the core integration tests (add-candy/rm-candy round-trips) stay in
  `charly/scaffold_project_test.go`, rewired to call `kit.ScaffoldProject`/`kit.AddBox` for setup.

## sdk pointer

Bumps the `sdk` submodule to the merged `feat(kit)` scaffold-engine commit (cross-repo order: sdk
first). No `charly.yml` schema-version change.

## Verification

`go build` + full `go test ./charly/...` + `./sdk/kit` green; `golangci-lint run` 0 issues (both
modules); `gofmt`/`golangci-lint fmt` clean; R5 grep clean (no core reference to the moved engine
funcs but `kit.*`). Live smoke on a fresh binary: `charly box new project`, `charly box new candy`
(candy `version:` CalVer stamped), and `charly box new box … --base … --candies …` (node-form image
written) all pass through the kit engine — byte-parity with the prior core behaviour.

# 2026.193.1830 — box-corpus tests are hermetic on a box-submodule-less checkout

`TestCandySourceDirs_OverrideAnchorsRemoteApk` and `TestCueBox_Corpus` (both in `charly/`) iterate
the `box/<distro>` submodule corpus. On a checkout where the box submodules are NOT inited (a dev
who clones charly + sdk to work on core but does not pull the five distro repos), both HARD-FAILED,
so `go test ./...` was red through no fault of the code under test.

## Root cause

An un-inited git submodule leaves an **empty gitlink placeholder directory**, not an absent path:

- `TestCandySourceDirs_OverrideAnchorsRemoteApk` guarded on `os.Stat(box/cachyos)` — which SUCCEEDS
  on the empty placeholder, so the skip never fired and `LoadUnified(box/cachyos)` then hard-failed
  (`ok=false`) on the missing `charly.yml` manifest.
- `TestCueBox_Corpus` globbed `../box/*/box/*/charly.yml` → empty → `ok==0` →
  `t.Fatal("no box entities validated")`.

`TestCueKinds_Corpus` was already hermetic (it reads `../charly.yml` + the embedded `charly.yml`
and `continue`s past unreadable box files), so it is unchanged.

## Fix — detect "checked out" by the MANIFEST, not the dir

One shared helper (R3) encodes the correct signal:

```go
// boxSubmoduleCheckedOut reports whether the box/<distro> submodule rooted at boxDir is
// checked out — signalled by its top-level charly.yml manifest, since an un-inited submodule
// leaves only an empty gitlink placeholder dir (os.Stat(boxDir) is a false positive).
func boxSubmoduleCheckedOut(boxDir string) bool { … os.Stat(filepath.Join(boxDir, "charly.yml")) … }
```

- `TestCandySourceDirs_OverrideAnchorsRemoteApk` skips when `boxSubmoduleCheckedOut(box/cachyos)` is
  false.
- `TestCueBox_Corpus` skips on an empty glob ONLY when NO box submodule is checked out
  (`filepath.Glob("../box/*")` ∩ `boxSubmoduleCheckedOut`); if a box submodule IS present yet the
  discovery glob matched nothing, it falls through to the existing `ok==0` `t.Fatal` — so a genuine
  glob/path regression still fails loudly. Skip is granted ONLY when the corpus is genuinely absent.

## Verification (both directions, live)

- Box-LESS (box/* not inited): `TestCandySourceDirs…` + `TestCueBox_Corpus` **SKIP** gracefully,
  `TestCueKinds_Corpus` PASS; full `go test ./... -count=1` **green** (54s) — no other box-dependent
  hard-failure remains (all other box-referencing tests use comments / ref-strings / temp fixtures,
  not the real corpus).
- Box-INITED (all five submodules checked out): both tests **RUN and PASS** —
  `TestCandySourceDirs…` resolves the committed APK via 134 candy source dirs; `TestCueBox_Corpus`
  validates 79/79 discovered box entities; full `go test ./... -count=1` **green** (57s). The fix
  skips only when genuinely absent; it never vacuously skips a present corpus.
- `golangci-lint run` 0 issues; `gofmt` clean; `golangci-lint fmt --diff` clean.

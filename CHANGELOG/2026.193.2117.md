# 2026.193.2117 — pre-commit-gate: read the tier from -F message files; unscannable forms fail CLOSED (#35)

The mechanical half of #35 — a follow-up to #34, found while landing it (surfaced by T-P8b). The
gate parsed the attribution tier ONLY from the `git commit` command string, so an external
`-F <file>` (my own #38/#83 commit form) or a `$(...)` substitution HID the tier, and all three
tier-dependent checks (attribution-absence, docs-only-diff, the #34 CHANGELOG requirement) silently
failed OPEN. `analysed on a live system` — the changed gate was executed live (`gate_test.py` 77/77
+ an old-vs-new differential proving both holes now BLOCK).

- **`.claude/hooks/pre-commit-gate.sh`** —
  - `scan_commit_args` now CAPTURES `-F`/`--file` values into `msg_files` and flags message-REUSE
    forms (`-c`/`-C`/`--reuse-message`/`--reedit-message`/`--fixup`/`--squash`, `REUSE_OPTS`).
  - New `read_file_tiers()` reads the trailer from a readable regular `-F` file (capped, never a
    device/FIFO); a `-` value is stdin (readable only via a heredoc, whose body is already in the
    command string — a piped `-` is unreadable).
  - The tier now comes from `TIER_RE.findall(cmd) + file_tiers` (shared `TIER_RE`, R3). When no tier
    is found and the message is not inherited (amend/reuse), the gate fails CLOSED on an unreadable
    message (`$(...)`/backtick, piped/unreadable `-F`, or an editor message) and reports absence on a
    readable one — uniformly for every tier-dependent check.
- **`.claude/hooks/gate_test.py`** — the old `-a -F msg.txt -> ALLOW` case (the documented fail-open)
  becomes `-> BLOCK` (fail-closed), plus 9 new cases: `-F <file>` tier read (no-CHANGELOG → BLOCK,
  CHANGELOG → ALLOW, docs-tier-on-code → BLOCK, no-trailer → BLOCK), nonexistent-`-F` → BLOCK,
  `$(...)` → BLOCK, heredoc-still-scannable → BLOCK, bare-editor → BLOCK, `--amend --no-edit` → ALLOW.
  77/77 green.

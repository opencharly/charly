# 2026.195.0330 — R44 — check-box persistent-container box-mode (exit-127 store-contention fix) (breaking!)

Fixes task #44: `charly check box` probes failed **exit-127** under concurrent image-build churn. RCA (the
verbatim evidence: `creating temporary passwd file for container … / Cleaning up container: unmounting
container`): the per-step `podman run --rm` container's rootfs mount + temp-`/etc/passwd` generation (the
image `USER=1000` path) raced the builders' layer commits in the shared podman store; the container setup
failed BEFORE the check ran, and charly laundered podman's OWN infra exit into a check FAILURE. Bumps the sdk
submodule to its R44 half (classification + the disposable-box-executor removal).

## Persistent-container box-mode (Option A — the root fix)

- `charly/check_box_container.go` (NEW) `checkBoxContainerChain`: `charly check box` creates ONE container
  (`podman run -d --pull=never --entrypoint= <img> sleep infinity`), runs every check step via `podman exec`
  into it (`ContainerChain`), then a single `rm -f` teardown — **O(N)→O(1) container setups** (the passwd-race
  surface), unifying box-mode with `charly check live`'s exec model. `host_build_check_run.go`
  (`hostCheckRunBox` + `hostFeatureBox`) switched off the deleted `ImageChain`; the ONE container-CREATE
  failure is classified (`kit.ClassifyContainerInfraFailure`) → the INFRA exit class.
- `candy/plugin-check/{check,live,feature}_cmd.go`: ONE shared `failErrorFor` (R3) — an infra-only run exits
  the INFRA class (**exit 1**, a plain error), a genuine check failure exits checks-failed (**exit 2**), a
  mixed run exits 2 (a real failure dominates). Never a pass-by-exhaustion; residual infra events stay LOUD.

## Semantic guard (Option A)

ONE persistent container shares per-step state (was: a fresh container per step). Doctrinally safe — `check:`
steps are idempotent goss-style probes, and `charly check live` ALREADY runs them against one shared
container; box-mode merely aligns. Verified live: `check box check-sidecar-pod` byte-identical (12 steps, 10
passed, 0 failed, exit 0), clean teardown.

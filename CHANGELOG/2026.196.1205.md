## Fix 1 — the ZERO-ALIASES hook gate (`pre-commit-gate.sh` + `gate_test.py`)

`pre-commit-gate.sh` gains `alias_gate(repo)`, a deterministic PreToolUse backstop
for the "Core is a PLUGIN HOST" ZERO-ALIASES standing rule:

- a NEW `charly/*_aliases.go` file (git status `A`) is BLOCKED outright — alias
  files have NO migration exception;
- a declaration-form kit-alias line in ANY staged `charly/*.go`
  (`type X = kit.Y` / `var x = kit.Y`) is BLOCKED;
- a grouped-block alias line (`    Name = kit.Y`) in a GROWN `charly/*_aliases.go`
  is BLOCKED.

A plain residual kit IMPORT or CALL is ALLOWED — only alias FORMS block
(IMPORT-PURITY of a plain import/call stays validator-judged). Fail-open for
non-charly repos (an sdk/plugins submodule leg with no `charly/` dir). The gate is
tier-independent. `gate_test.py` gains 8 ZERO-ALIASES cases + a `charly_repo()`
helper (a charly-superproject shape with no `go.mod`, isolating the alias gate
from the lint gate): NEW alias file blocked (#86 class), declaration-form var/type
alias blocked, grouped alias in grown alias file blocked (#87 class), plain kit
CALL allowed, plain kit IMPORT allowed, alias outside `charly/` not gated, clean
charly change allowed. **32/32 cases PASS** — the changed `alias_gate` code path
executed LIVE against real staged `charly/*_aliases.go` files + real `git diff`.

## What moved (→ candy/plugin-authoring, IMPORT-PURE: imports ONLY github.com/opencharly/sdk)

- `charly box set/add-candy/rm-candy/write/cat` — the 5 PURE authoring verbs. They run entirely on
  sdk/kit (`kit.SetByDotPath` / `kit.MappingChild` / `kit.SaveYAMLNodeFile`) + stdlib + the moved
  `addCandyToBox` / `removeCandyFromBox` yaml.Node helpers + the moved `resolveProjectFile`
  path-traversal guard. ZERO core reentry.
- `charly box fetch/refresh` — the remote-repo cache pre-primer / force-re-clone. The repo resolver
  (`ResolveProjectRepo` → `EnsureRepoDownloaded`: `CHARLY_REPO_OVERRIDE` + the refs-backend dispatch
  + the command:migrate auto-migration) is HOST-COUPLED, so the plugin re-runs the NEW hidden core
  `__box-fetch` / `__box-refresh` reentry over `HostBuild("cli")` — the SAME seam candy/plugin-box's
  `pkg` verb uses for `__box-pkg`.

## What stays core (fabric)

- `charly/box_fetch_reentry.go` (NEW, ~70 LOC) — the hidden `__box-fetch` / `__box-refresh` reentry
  structs (`BoxFetchCmd` / `BoxRefreshCmd`), registered as hidden Kong commands in `main.go`. K5-doomed:
  both die when the repo resolver moves into a sdk kit the plugin can consume directly.
- The CLI/prescan fabric (`collectExternalCommandPlugins` nests the 7 command words under `box` via
  `CommandParent()=="box"`; `dispatchInProcCommand` routes `Invoke(OpRun)` in-proc).
- `ResolveProjectRepo` / `normalizeRepoSpec` / `EnsureRepoDownloaded` (host-coupled repo resolution,
  load-bearing for `--repo` + remote-candy fetch).

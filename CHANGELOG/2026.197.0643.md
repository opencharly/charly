# 2026.197.0643 — #52b sub-batch 2: repoint 6 more vmshared spec-passthrough types to spec

Second sub-batch of the vmshared→spec envelope-route repoint (task #16). Repointed 6 more pure
`= spec.X` passthrough symbols directly to `spec.X`, deleted their `vmshared_aliases.go`
bindings: `DistroDef` → `spec.ResolvedDistro`, `VmDeployState`, `Matcher`, `MatcherList`,
`PortSpec`, `CandyYAML`.

## Three new classes of mechanical-rewrite bug found and fixed (beyond sub-batch 1's method-call
## collision)

1. **Embedded/named struct field sharing the type's old bare name.** `InlineCandy` embeds
   `spec.CandyYAML` anonymously; a composite literal for an anonymous embedded field must key on
   the field's PROMOTED name (`CandyYAML`), never the qualified type
   (`&InlineCandy{spec.CandyYAML: c}` is invalid Go). Same issue for `ResolvedBox.DistroDef` — an
   ordinary (non-embedded) field literally named `DistroDef`, so `&ResolvedBox{DistroDef: ...}`
   must keep the bare field-name key even though the field's TYPE is now `*spec.ResolvedDistro`.
   8 call sites across `distro_cascade_test.go`/`localpkg_test.go`/`tasks_test.go` fixed
   (`spec.ResolvedDistro:` → `DistroDef:`, value-side references left qualified); 2 call sites
   in `node_candy.go`/`provider_kind_invoke.go` (`spec.CandyYAML:` → `CandyYAML:`). `go build` /
   `go vet` caught every instance (`invalid field name` / `not a type`).
2. **Local identifier literally named `spec` shadowing the package import.** `spec *VmSpec` is
   the established parameter name throughout the VM build/bootstrap code
   (`vm_bootstrap.go`/`vm_build.go`/`vm_bootc_install.go`/`vm_cloud_image.go`). Introducing
   `spec.VmDeployState`/`spec.ResolvedDistro` as ANOTHER parameter's type in the SAME function
   made every `spec.Source`/`spec.DiskSize`/… body reference ambiguous (`spec.ResolvedDistro is
   not a type` — Go resolved `spec` to the local `*VmSpec` value, not the package). Fixed by
   renaming the shadowing local identifier to `vmSpec` throughout each of the 8 affected function
   bodies (`BuildBootstrapVM`, `buildBootstrapRootfs`, `buildBootstrapDisk`,
   `buildBootstrapSeedISO`, `runVmSpecBuild`, `BuildBootcVM`, `BuildCloudImage`,
   `RegenerateSeedISO`) — package-qualified `spec.X` references left untouched.
3. **Go's `...Type` variadic ellipsis false-matched as a selector dot.** Sub-batch 1's fix for
   collision class 1 above (never qualify an occurrence preceded by `.`) had a gap: `specs
   ...PortSpec` is preceded by THREE dots (the variadic ellipsis), not a selector — the rewriter
   incorrectly treated the third dot as `x.Field` and skipped it, which `go vet` caught
   (`undefined: PortSpec`, `config_test.go`). Fixed the one occurrence directly and hardened the
   rewrite tool (checks for a 3-dot ellipsis before excluding); re-scanned the full diff for any
   other silently-skipped `...Symbol` occurrence across all 19 symbols repointed so far — none
   found.

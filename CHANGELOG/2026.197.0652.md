## The known collision, handled up front this time

Proactively checked for method/field/local-var collisions BEFORE running the mechanical rewrite
(the discipline sub-batch 2's three bug classes motivated). Found the expected one:
`uf_box_generic.go` declares two accessor methods literally named `BoxConfig` ‚Äî
`(uf *UnifiedFile) BoxConfig(name string) (BoxConfig, bool)` and
`(c *Config) BoxConfig(name string) (BoxConfig, bool)`. Handled that ONE file by hand: the
method NAMES stay `BoxConfig` (unqualified ‚Äî Go requires it, they're declarations, not type
references), their RETURN TYPES become `spec.BoxConfig`, and every OTHER bare `BoxConfig`
reference in the file (helper functions `decodeBox`/`encodeBox`/`boxConfigFrom`/`eachBox`, none
of which are named `BoxConfig` themselves) repoints normally. The other 26 files' ~30 occurrences
are all `.BoxConfig(...)` accessor CALLS on `UnifiedFile`/`Config` ‚Äî correctly left untouched by
the rewrite tool's preceding-dot exclusion (these are method calls, not type references).

## A tooling bug of my own making, caught and fixed before it shipped

The batch script's exclusion list for "already hand-fixed, don't reprocess" compared the RAW
`git grep -l` output (`./uf_box_generic.go`, with the `./` prefix) against a plain
`("vmshared_aliases.go", "uf_box_generic.go")` tuple ‚Äî the comparison never matched, so BOTH
`uf_box_generic.go` (already hand-fixed) and `vmshared_aliases.go` (the alias file itself, which
also textually contains "BoxConfig" in its own declaration line) got reprocessed by the blind
batch replacer. Caught immediately by `go build` (`expected '(', found '.'` ‚Äî the tool had
turned the two method names into `func (uf *UnifiedFile) spec.BoxConfig(...)`, invalid Go, and
`vmshared_aliases.go`'s own `BoxConfig = vmshared.BoxConfig` declaration into
`spec.BoxConfig = vmshared.BoxConfig`, also invalid). Reverted both by hand (re-applying the
manual method-name fix; deleting the now-doubly-processed alias line, which needed deleting
anyway), removed the erroneously-added `spec` import from `vmshared_aliases.go`, and rebuilt
clean.

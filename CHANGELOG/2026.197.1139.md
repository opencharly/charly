## R1 finding surfaced while sweeping alias_collect.go (fixed in the same edit)

The file's own comment claimed `aliasNameRe` was "the LOAD-TIME validation
regex (validate.go rejects malformed alias names)". Verified FALSE: `validate.go`
has no alias-related code at all, and `aliasNameRe` in charly/core has no
production call site anywhere (only its own unit test exercises it).
`candy/plugin-box/validate_rules.go` carries an INDEPENDENT copy of the same
regex that IS wired into `charly box validate` — the box-authoring validation
externalization moved the real enforcement there without sweeping this now-orphaned
core copy. Corrected the comment to state the truth rather than delete the dead
code outright (deleting it would also drop `alias_collect_test.go`'s
`TestAliasNameRegex` pattern-regression coverage unless mirrored into the
plugin first, which is out of this cutover's doc-only K-annotation scope) —
flagged for a future dead-code batch instead.

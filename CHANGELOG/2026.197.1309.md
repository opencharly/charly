## Audit method (the lighter form, per revised guidance — gofmt -r is NOT a reliable AST-safety
oracle for composite-literal keys/locals)

Chose the lighter audit over full per-hunk classification because the risk category
(under-qualified bare references to a deleted alias) is a **guaranteed hard compile error**, not
a silent pass — there is no scenario where a missed repoint site both (a) fails to qualify and
(b) still compiles, since package main no longer declares any of the 23 old bare names at all.
`go build`/`go vet`/`go test`, clean across the *entire* merged tree (`charly` + every touched
plugin module: `plugin-vm`, `plugin-box`, `plugin-build`, `plugin-loader`, `plugin-tunnel`), is
therefore a complete proof for that risk class — it is exactly what caught the one real bug
above. Supplemented with a proactive grep for the ONE risk class a compiler can't catch (a
rewrite accidentally landing inside a string literal instead of code) across every touched
non-test file — found only legitimate prose/real-code matches, no corruption.

**Batch A (3 already-aliased symbols, zero new schema work):**
- `charly/unified_targets.go`: `deploykit.DeployExecutor` → `spec.DeployExecutor`
  (`runUnifiedTargetChecks`'s `exec` parameter). Already a Go type alias
  (`deploykit.DeployExecutor = spec.DeployExecutor`, `sdk/deploykit/steps.go`).

  → `spec.RouteConfig` (both already-existing aliases) at the two

  untouched — same-type aliases, so the assignment compiles unchanged;
  repointing the field declarations themselves is `layers.go`'s own,
  much larger K1-gated scope, not part of this chunk.

**Batch F consumer repoint:** `charly/host_build_render_seam.go`'s 4
render-seam dispatch cases now decode `spec.InlineBuilderParams`/
`spec.LocalPkgParams`/`spec.EnsureBuildersParams`/`spec.EmitPluginOpParams`
and construct `spec.InlineBuilderResult`/`spec.LocalPkgResult`/
`spec.EmitPluginOpResult` directly (the underlying sdk-side CUE-sourcing

for other, genuinely-behavioral reasons (`CandyModel`, `CompileLocalPkgStep`,
`LocalPkgInstallStep`, `HostContext`) and `buildkit` for `ResolvedBox`
(via `Generator.Boxes`, itself K1-gated) — unrelated to this specific
type family, so the file does not retire from the metric via this chunk.

**Batch B — DROPPED, not viable as scoped.** The 3 files originally
proposed for a `buildkit.ResolvedBox` → `spec.ResolvedBoxView` repoint
were re-verified against the actual source and ALL THREE are disqualified,
for three DIFFERENT reasons:
- `namespace.go` calls `ri.Builder.AllBuilder()` — a genuine method call
  on `buildkit.BuilderMap` (a field of `ResolvedBox`), not a field read.
- `builder_preresolve.go` reads `img.BuilderConfig.Builder[bName]` —
  `BuilderConfig` is not a field on `spec.ResolvedBoxView` at all; this
  is really the DistroConfig/BuilderConfig mechanism dependency (a
  DIFFERENT, separately-tracked escalation), not a simple ResolvedBox
  field-read.
- `host_build_render_seam.go` reads only `img.Tags` (which DOES exist on
  `ResolvedBoxView`), but `img` itself is typed `*buildkit.ResolvedBox`
  because it comes directly from `Generator.Boxes map[string]*buildkit.ResolvedBox`
  (`charly/generate.go`) — a field whose TYPE cannot be repointed without
  a much larger, out-of-scope change to `generate.go` (itself K1-gated).
  Merely reading a plain field off a value fetched from a `buildkit`-typed
  map still requires importing `buildkit` for the variable's type, even
  with zero method calls.

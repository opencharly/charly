# 2026.198.0910 — W3 build-engine cone: localpkg + intermediates leave core (placeholder, evaluator finalizes)

Wave W3 of the core-minimization program: the build-engine's LocalPkg render
seam and the auto-intermediate-image subsystem relocate out of `charly/` core
into `sdk/deploykit`+`sdk/buildkit`, after an RDD spike proved their claimed
host dependency was stale. The remaining registry-coupled render-seam methods
(EmitPluginOp / EnsureBuilders / InlineBuilder) and the loader-coupled files
(`layers.go`, `generate.go`'s `NewGenerator` resolve half, `format_config.go`,
and the resolve-side quintet — `render_baked_metadata.go`/`security.go`/
`hooks.go`/`layer_capabilities.go`/`layer_secrets.go`) are OUT of scope for
this cutover — the first is a permanent kernel M-mechanism citizen (the
provider registry), the second re-buckets to the resequenced W9/K1 loader
wave, both confirmed by spike, not by trusting the pre-existing header claims
(several of which were independently found stale during this cutover's own
audit).

## What moved (full detail: sdk `CHANGELOG/2026.198.0900.md`)

- **LocalPkg**: `charly/localpkg.go` (559 LOC) deleted whole; its logic +
  `renderBuilderScript`/`hostBuilderContext` (carved out of
  `charly/deploy_host_helpers.go`, deploy-side functions there untouched) move
  to `sdk/deploykit/localpkg.go` as pure functions. The `LocalPkg` render-seam
  HostBuild round-trip is eliminated entirely (traced: the caller already held
  the fully-populated step). All call sites repointed
  (`builder_venue.go`/`pkg_cmd.go`/`plugin_executor_reverse.go`/
  `step_emit_hostbuild.go`/`tasks.go`); tests split between the pure half
  (moved to `sdk/deploykit/localpkg_test.go`+`builder_scripts_test.go` repoint)
  and the core-coupled half (kept in `charly/localpkg_test.go`, calling the
  relocated `deploykit.*` functions).
- **Intermediates**: `charly/intermediates.go` (454 LOC) +
  `charly/intermediates_test.go` (1379 LOC) +
  `charly/intermediates_determinism_test.go` (155 LOC) deleted whole; the
  host-coupled compute half moves to `sdk/deploykit/intermediates_compute.go`
  + a new shared `sdk/buildkit/distro_builder_map.go` (an R3 fix: charly's own
  `Config.distroBuilderMap` now shares the same lookup algorithm instead of a
  second copy). `charly/intermediates_shim.go` shrinks 65→42 lines (the one
  remaining closure lifting `cfg.Defaults` into the new
  `deploykit.IntermediateDefaults`). Full test migration onto
  `CandyModel`/`NewSpecCandyModel` fixtures.

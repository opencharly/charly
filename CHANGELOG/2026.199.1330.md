# 2026.199.1330 — federated agent channel foundation

- Add generic `agent-runtime` and `terminal` provider classes and a hidden
  `__agent-target serve --stdio` endpoint. Nested local/SSH gRPC routes consume
  one route segment per Charly node; tmux remains provider data above gRPC.
- Replace the recursive `command:tmux` shell-back with `terminal:tmux` and
  `agent-runtime:tmux`. Control mode supplies acknowledged ordered raw
  output, format subscriptions supply deterministic pane exit state, and input
  uses literal argv, allowlisted keys, buffers/paste, and explicit resize frames.
- Add the ephemeral native Pi SDK runner and `agent-runtime:pi` adapter pinned to
  Pi 0.80.10. It reuses `createAgentSessionRuntime`, `SessionManager`, and Pi's
  upstream `runRpcMode`; no permanent service is installed.
- Add `kind:agent-team` and CUE-generated team topology records.
- Add the daemon-free `charly agent` control plane: durable UUIDv7 sessions,
  atomic idempotent runs, sequenced events, follow-up/steer/delegation, typed
  terminal operations, team authority, SSH federation metadata, and mandatory
  incident/RCA/recovery decisions. MCP reflection marks every mutating leaf.
- Add candy-authored Claude Code, Codex, and Gemini terminal profiles, a Pi-TUI
  controller, and an opt-in adapter using Pi orchestrator's official CLI.
- Reuse established Go components at the implementation boundaries:
  Charmbracelet `x/vt` and `x/ansi` own terminal emulation, `x/term` owns the
  controller TTY, `google/uuid` owns UUIDv7 generation, `fsnotify` owns
  cross-process abort notifications, and `renameio` owns atomic record
  replacement. JSONL remains the CUE-record evidence format and is file- and
  directory-synced after each mutation.
- Retain the narrow tmux control-mode adapter after evaluating `gotmuxcc`,
  Guppi, `owenthereal/tmux`, and Stapler Squad: none combines an externally
  consumable raw-output subscription, extended-output support, isolated
  per-run sockets, compatible licensing, and a reusable lifecycle API. The
  adapter delegates sessions, panes, PTYs, history, and input to tmux itself;
  it only maps tmux's documented control stream into the generated channel.
- Preserve `charly tmux` as a compatibility command model backed entirely by
  `terminal:tmux`. Its named sessions are durable agent session records and its
  operations use typed input/snapshot/close calls, so compatibility does not
  reintroduce the former default-socket or recursive shell-back implementation.
- Keep OpenSSH as the outer process transport so user SSH configuration,
  agents, host verification, and jump hosts retain native behavior. gRPC-Go's
  custom dialer carries HTTP/2 over that full-duplex process, while the small
  single-connection listener is the server-side stdio bridge. Target hops stay
  generic CUE data, so deployment→gRPC→tmux, SSH→gRPC→tmux, and recursively
  nested routes share the same implementation.
- Keep image-owned Claude Code installs pinned across live terminal sessions by
  disabling the upstream background updater at the candy runtime boundary.
- Render pac, rpm, and deb local-package builds against the SDK's isolated
  package-source copy while retaining explicit read-only access to the current
  project inputs; Arch `pkgver()` can no longer rewrite the tracked PKGBUILD.
- Give every external go-plugin client one Charly-owned WARN-level logger so
  ordinary gRPC shutdown EOF traces stay quiet while real plugin warnings and
  errors remain visible to operators.
- Inventory Git-backed local-package inputs as tracked plus non-ignored files,
  excluding stale package-manager caches and VCS mirrors from every isolated
  build while retaining the filesystem path for non-Git sources.
- Mark terminal channels with their actual launch, attach, snapshot, control,
  or run operation. A successful control mutation can no longer claim and
  remove the run-owned tmux server before its waiting controller observes exit.

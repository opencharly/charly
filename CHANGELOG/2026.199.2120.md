# 2026.199.2120 — Kimi harness integration: generic gate scripts, kimi profile parity

The repository gains Kimi Code as a third supported agent harness.

Gate and reminder scripts in `.claude/hooks/` are now harness-generic: their
block reasons and second-pass reminders cited `CLAUDE.md` as the sole rulebook,
but the scripts are payload-compatible with any harness (Kimi hooks delegate to
them) and both rulebooks carry identical R0–R10 rules. Messages now cite the
rule/section with "the project rulebook" wording; `BLOCKED:` formats, exit
codes, and behavior are byte-identical (proven by the payload matrix +
`gate_test.py`).

`scripts/validate-agent-config.py` now drift-checks the kimi developer profile
alongside claude and codex (`plugins/setup kimi --check developer`), so the
third harness cannot silently fall out of sync.

`AGENTS.md` gains a "Kimi project configuration" section: Kimi reads this
rulebook and the repo-native `.agents/skills/` tree natively; permission rules
and hooks live in the user-level `~/.kimi-code/config.toml` (merged from the
`kimi-user-config.toml` snippet, repo-guarded to OpenCharly checkouts); fresh
teammates/validators run as separate `kimi` sessions (no custom subagent
registry) under the unchanged fresh-validator protocol. `README.md` documents
the third harness.

Lands with the `plugins` gitlink pinned to the merged plugins main that
provides `plugins/setup kimi` (the kimi harness mode, the snippet, and the
harness-generic rulebook citations across the skill tree).

Assisted-by: Kimi Code Moonshot Kimi K3 (fully tested and validated)

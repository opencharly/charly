# 2026.200.0000 — package install-test matrix: 7 distro cells, one canonical harness

`scripts/pkg-install-test.sh` is the new single canonical definition of the
native-package test matrix (R3): `task pkg:test` runs it locally and the
`release-packages` workflow calls the SAME script per format after each build —
no second inline copy in YAML. Each cell installs the freshly built artifact in
a throwaway rootless-podman container and asserts the package manager
auto-resolves every mandatory dep from the distro's own repos, `charly version`
equals the build CalVer, and the welded command plugins dispatch project-less.

The matrix now covers every distro version FULLY supported by the upstream
maintainers (verified 2026-07-18 against endoflife.date): arch + cachyos
(rolling, cachyos via this repo's own `box/cachyos` base pin — never a
duplicated digest), debian:13, ubuntu:24.04, **ubuntu:26.04** (new cell —
26.04 LTS released 2026-04-23), fedora:43, and **fedora:44** (new cell —
released 2026-04-28; fedora 42 went EOL 2026-05-27, debian 12 left
security-team support 2026-07-11, so neither gets a cell). A failing cell fails
the workflow BEFORE the tag-guarded release upload — previously the artifacts
were built and released with zero install testing, so dep drift on a new distro
release would have shipped green. The stable-named `opencharly-amd64.*` release
alias copies are made only AFTER the gate: the script's artifact globs must
match exactly one file per format (a canonical + alias pair is a duplicate
target for `pacman -U` / a same-NEVRA pair for `dnf` — caught by the fresh
pr-validator's CI-layout replication, fixed by the step reorder).

RDD spike result (the matrix's first full run, artifacts at CalVer
2026.199.2318): ALL 7 cells PASS — including ubuntu:26.04 and fedora:44, so the
existing .deb/.rpm already install cleanly on the newest maintainer-supported
releases with zero dep drift. The spike surfaced and root-fixed two harness
bugs (an `&&`-chain redirect binding only the last command, and a per-probe
fresh-container design that re-downloaded the dep closure every probe — the
canonical script runs ONE container per cell).

`taskfiles/Build.yml` gains `pkg:test` plus the version-bump runbook (when a
distro revs: check upstream support, add/drop the cell in the script, RCA any
dep-name drift into the owning `pkg/<fmt>` repo, keep the one-venue-per-format
build container deliberate) and a stale comment fix (rpm/deb builds run in
rootless podman, not directly on the host). The cell list in this matrix is
package install+run support only; box-substrate support for ubuntu 26.04 /
fedora 44 (new base entities, `distro.*.version` bumps, beds) is a separate
cutover — fedora 43's 2026-12-09 EOL is its deadline.

# 2026.200.1707 — fix(candy): xurl — run the postinstall npm 12 blocks

The builder image carries npm 12, which blocks package install scripts unless
they are explicitly allowed. `@xdevplatform/xurl`'s postinstall (`install.js`)
is what downloads the real platform binary, so the global install completed
but `xurl --help` failed with `xurl binary not found` (surfaced by the
check-openclaw-full-pod / check-openclaw-desktop-pod beds, 2/171 and 2/355
failures respectively).

The plan now runs the package's own download script explicitly after the npm
install — one audited script, no global allow-scripts policy change, no
suppression. Proven live on npm 12.0.1 (the builder image's exact version):
`node .../install.js` fetches the binary and `xurl --help` succeeds.

Sibling note: other npm candies with postinstall-bearing deps (gemini's
`protobufjs` / `@google/genai`) have optional scripts and still pass; the
shared builder-template policy for the whole class belongs to the named
bed-image-hygiene batch.

# 2026.201.1002 — Delegate mutable repo refs to the freshness-checked downloader (@main skew root fix)

`EnsureRepoDownloaded` reused any cached repo export without consulting the
downloader: a mutable ref (a branch such as `main`, or an unversioned ref
resolved to the default branch) froze at its first-download content. A
pre-#146 `charly@main` export therefore kept serving protocol-v1
builder-plugin sources, and box builds with unversioned builder-plugin refs
failed `Plugin version: 1, Client versions: [2]` (issue opencharly/charly#158)
even after the box repos re-pinned their candies to the protocol-v2 tag —
the pin is the candy fetch coordinate, but the out-of-process builder plugins
resolve by canonical unversioned ref, so the reconcile alone could not clear
the skew.

Mutable refs now always delegate to the registered refs backend: the
downloader re-resolves the ref's current commit and refreshes a stale export
(the sdk `kit.DownloadRepo` provenance contract, producer leg
opencharly/sdk#77). Immutable coordinates (tags, SHAs) keep the offline
cache-hit fast path. The mutable/immutable decision uses the new shared
`kit.IsMutableRef` classifier.

Verified: `TestEnsureRepoDownloaded_MutableRefAlwaysDelegates` (stub
downloader — mutable delegates when cached, tag hits offline),
`TestIsMutableRefCoreContract`; core Go gate green (`go test ./...`,
`go vet`, `golangci-lint` 0 issues); live end-to-end on the rebuilt binary:
a stale `charly@main` export self-healed on first access and
`charly -C box/arch box build arch-test` (the issue's repro) passed.

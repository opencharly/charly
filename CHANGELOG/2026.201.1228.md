# 2026.201.1228 — DEPLOY wave: pod-lifecycle CLI port to candy/plugin-pod

The 11 pod-lifecycle CLI commands (`start`/`stop`/`restart`/`logs`/`remove`/`shell`/
`service`/`volume`/`cp`/`config`/`update`) move out of hardcoded core Kong structs
into a new compiled-in plugin candy, `candy/plugin-pod`, using three seam shapes
proven live on the built binary:

- **Zero-seam pure-logic move** — `VolumeCmd`/`CpCmd` (+ their List/Reset leaves)
  carry no host/registry/loader coupling and moved wholesale, zero seam.
- **Per-command HostBuild seam** — `start`/`stop`/`logs`/`remove`/`shell`/`update`
  each keep a thin Kong grammar in the candy that forwards authored flags via
  `sdk.Executor.HostBuild(ctx, "pod-<word>", reqJSON)` to a host-registered builder
  (`charly/host_build_pod_*.go`) that reconstructs the original core orchestration
  struct (renamed `pod<Word>Cmd`, header-noted "TRACKED P13-KERNEL EXIT") and runs
  its unchanged `.Run()` body — needed because that body holds the provider
  registry and/or project loader, which a plugin cannot hold.
- **Operation-discriminated shared seam** — `service`/`config` collapse their
  former 4-/6-way dedicated-CommandProvider shims into one Kong leaf group each,
  dispatching through a single `HostBuild` seam keyed by an `operation!:`
  discriminator on the CUE-sourced wire request.

Companion sdk-repo leg (opencharly/sdk#78, merged as `v0.2026201.1135`) added the
13 CUE-sourced wire-type pairs these seams carry, plus 2 genuine relocations
completing prior waves' incomplete cutovers: `sdk/kit/quadlet_paths.go` (pod
quadlet path helpers) and `sdk/deploykit/pod_service.go` (pod service
start/stop/restart bodies).

While assembling the wave, an R1 archaeology sweep found and completed 4
incomplete cutovers from earlier waves whose bare core duplicates had never been
swept: `charly/provides.go` (duplicate of `sdk/deploykit/provides.go`),
`charly/agent_forward.go` + its test (duplicate of `sdk/kit/agent_forward.go`),
and `charly/quadlet.go`/`quadlet_pod.go` + their tests (relocated to
`sdk/kit/quadlet_paths.go`) — all deleted, every caller repointed to the sdk
equivalent. The sdk leg's own validator round then surfaced (and this wave's
producer fixed) 2 header comments asserting a completed migration that grep
disproved — corrected to state the true pre-merge relationship, sweeping 2 more
identical instances beyond the one flagged.

11 now-obsoleted dedicated-CommandProvider shim files (`plugin_command_*.go`,
superseded by the new candy's single provider) are deleted in the same cutover.

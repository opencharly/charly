### The CheckResult wire envelope (the operator-approved embed split, ruling R-FS2 + (A))

`spec.CheckResult` (CUE-sourced in the `sdk` submodule) replaces the former
package-main `CheckStatus`/`CheckResult`/`TestPass`/`TestFail`/`TestSkip`
aliases in `charly/checkrun.go` — DELETED entirely. Every registry-coupled
floor file that constructs or reads a check verdict now references
`spec.CheckResult`/`spec.StatusPass`/`spec.StatusFail`/`spec.StatusSkip`
directly: `provider.go`, `provider_verb.go` (the `CheckVerbProvider`
interface), `verb_builtins.go`, `unified_targets.go`, `provider_checkenv.go`,
`check_kit_adapter.go`, `planrun_adapter.go` (`hostVerbResolver` — now
satisfies `kit.VerbResolver`'s `spec.CheckResult`-returning signature),
`checkrun_act.go`, and `checkrun.go`'s own `passf`/`failf`/`skipf` helpers.
Zero new `sdk/kit` import lands on the floor.

**Breaking wire-format change** for `charly check box/live/run --format
json`/TAP consumers: the JSON keys rename `Op`→`op`, `Verb`→`verb`,
`Status`→`status`, `Message`→`message`, `Elapsed`→`elapsed` (a deliberate
consistency fix — the former hand-written type carried no json tag on these
fields, so encoding/json defaulted to the bare capitalized Go field name).
See the `sdk` CHANGELOG entry for the full key-mapping table and the golden
parity-test proof that this is the ONLY change (same field set, same values,
same omission behavior).

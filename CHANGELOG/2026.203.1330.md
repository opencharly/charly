# 2026.203.1330 ŌĆö SDD-conversion consumer fixes: GroupAccessible + VFIOGpu

Companion to `opencharly/sdk#92` (merged, `v0.2026203.1311`): the two
representation reshapes that PR's `HostProbeReply.GroupAccessible` and
`VFIOGpu` conversion required are wire-format-identical, but Go-source
breaking ŌĆö this cutover updates every consumer.

## What changed

- **`sdk` gitlink** bumped to `922b3469` (the merged `feat/sdd-wire-conversion`
  squash) + `charly/go.mod`'s `github.com/opencharly/sdk` require bumped to
  `v0.2026203.1311` (in-tree resolution rides `replace ../sdk`; the require
  version matters only for out-of-tree consumers, per convention).
- **`HostProbeReply.GroupAccessible`** is now `map[string]bool` (was
  `map[int]bool` ŌĆö CUE has no int-keyed-map construct; `encoding/json`
  already renders a `map[int]bool`'s keys as decimal strings on the wire, so
  this is a pure Go-source fix, zero wire-format change). Updated both real
  construction/read call sites: `charly/host_build_hostprobe.go` (builds the
  map, keys via `strconv.Itoa`) and `candy/plugin-doctor/command.go` (reads
  it, same key form).
- **`VFIOGpu`** no longer embeds `VFIOPCIDevice` anonymously (CUE unification
  has no Go-embedding concept; Go's own JSON-promotion of an embedded
  struct's fields already made the wire shape identical either way, so this
  is also zero wire-format change). Every `VFIOGpu{VFIOPCIDevice: d}`
  composite literal is replaced with the new `spec.NewVFIOGpu(d)`
  constructor: `candy/plugin-gpu/detect.go`, `candy/plugin-gpu/switch.go`
  (2 sites total, one per file) + test fixtures `candy/plugin-gpu/switch_test.go`
  (2 sites), `charly/gpu_prereq_test.go` (2 sites), `charly/gpu_allocate_test.go`
  (2 sites, restructured from a nested slice literal into two named
  `spec.NewVFIOGpu(...)` calls since the flattened struct can no longer host
  an embedded-field key inside a composite literal).
- **`candy/plugin-build/schema/build.cue`** ŌĆö one doc-comment citation of the
  now-deleted `sdk/spec/deploy_wire.go` repointed to `sdk/schema/buildwire.cue`
  (R5 sweep; this comment lives in the superproject, not the sdk repo, so it
  was out of scope for `sdk#92`'s own grep self-test).

`candy/plugin-vm/vmshared_aliases.go`'s dangling `ValidateLibvirtSnippet`
alias ŌĆö the OTHER pre-existing gap this cutover initially found blocking its
own local build/bed verification ŌĆö needed NO fix here: `opencharly/charly#179`
(the dead-code radical-removal batch, merged ahead of this branch) already
deleted that exact line. Verified absent on this branch's base (post-#179
main) before writing this cutover's scope.
